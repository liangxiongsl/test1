html

npm install -D tailwindcss
npx tailwindcss init

在根目录下生成的 ./tailwind.config.js 补充代码：

/** @type {import('tailwindcss').Config} */
module.exports = {
  content: ["./src/**/*.{html,js}"],
  theme: {
    extend: {},
  },
  plugins: [],
}

编辑 tailwind 的源文件 ./path/to/input.css：

@tailwind base;
@tailwind components;
@tailwind utilities;

编译 input.css => 生成 output.css

npx tailwindcss -i ./path/to/input.css -o ./path/to/output.css --watch

在 index.html 中引入 output.css

...

<link href="./path/to/output.css" rel="stylesheet">

...

核心

tailwind.css 使用类名的组合来修饰元素的状态

修饰符

伪类修饰符

<修饰符>:<类名>

对应伪类 伪类修饰符 描述 例子
:hover hover 鼠标悬停
:focus focus 聚焦 鼠标按下并松开后 / tab 聚焦
:focus-within focus-within 当前元素或其子类聚焦
:focus-visible focus-visible 用键盘聚焦元素
:active active 鼠标按下
:visited visited 已访问
:target target url #hash 与当前元素 id 匹配
:first-child first 首个
:last-child last 最后一个
:only-child only 当前元素是父元素的唯一儿子
:first-of-type first-of-type
:last-of-type last-of-type
:only-of-type only-of-type
:nth-child(odd) odd
:nth-child(even) even
:empty empty 元素没有内容 class="empty:hidden"
表单相关伪类 伪类修饰符 描述 例子
:required required
:enabled enabled 启用 用于 input 标签等
:disabled disabled 禁用
:valid valid
:invalid invalid
:checked checked 已勾选 用于 <input type="checkbox">
:indeterminate indeterminate 不确定状态 用于 <input type="checkbox">
:default default
:in-range in-range
:out-of-range out-of-range
:placeholder-shown placeholder-shown 占位符
:autofill autofill
:read-only read-only 只读

类 group 和 伪类修饰符前缀 group- => 父元素伪类状态影响子元素伪类状态
=> 例子：

<div class="group">
	<div class="group-hover:text-white">example text1</div>
	<div class="group-hover:text-white">example text2</div>
</div>

=> 父元素悬停时，子元素的文本颜色变为白色

类后缀 /<group-name> 和伪类修饰符后缀 /<group-name> => 用于标识组的 id（即起名组）
=> 例子：

<div class="group/g1">
	<div class="group-hover/g1:text-white">example text1</div>
	<div class="group-hover/g2:text-white">example text2</div>
</div>

=> 父元素悬停时，只有第一个子元素的文本颜色变为白色

类 peer 和伪类修饰符前缀 peer- => 兄弟伪类状态影响另一些兄弟元素伪类状态
=> 例子：

<div class="peer">example text1</div>
<div class="peer-hover:visible invisible">example text2</div>

=> 悬浮于第一个元素时，显示第二个元素
=> 再如：

<div>
	<button class="peer/t">下拉框</button>
	<ul class="invisible peer-hover/t:visible hover:visible">
	<li v-for="(v) in 5" class="hover:bg-cyan-500">{{v}}</li>
	</ul>
</div>

注：peer 只能对后面的兄弟起作用
注2：类似于起名组 group/<group-name> 和 group-<class>/<group-name>，起名兄弟 peer/<peer-name> 和 peer-<class>/<peer-name> 也是合法的

关联状态语法 类 关联类 补充
父子分组 group group-<class>
起名的父子分组 group/<name> group-<class>/<name>
匹配任意父亲组 group-[<filter>]
兄弟分组 peer peer-<class> 关联类对应的元素必须在后面
起名兄弟分组 peer/<name> peer-<class>/<name>
匹配任意兄弟组 peer-[<filter>]

*:<class> => 为当前元素的所有子元素设置类 => 例如：

<ul class="*:text-red-800">
  <li v-for="(v) in 10">{{v}}</li>
</ul>

注：*:<class> 优先于（或者说 会覆盖）其对应的直接子元素（不包含非直接的孙子元素）的重名类

has-[<filter>]:<class> => 为当前元素的部分子元素设置类
group-has-[<filter>]:<class> => 根据父元素的后代指定是否为当前元素设置类
peer-has-[<filter>]:<class> => 根据兄弟元素的后代指定是否为当前元素设置类
