安装、编译

css 预处理器 Sass 基于 ruby 开发，可通过 gem 安装：

gem install sass
gem install compass

三种 sass 的命令编译方式：

  • 单文件转换 => sass <input.scss> <output.css>
  • 单文件监听 => sass --watch <input.scss>:<output.css>
  • 侦听整个目录 => sass --watch <input/path>:<output/path>
    额外选项：
  • --style => 指定解析后 css 的排版格式 => nested, expanded, compat, compressed
  • --sourcemap => 开启 sourcemap 调试 => 会生成 .css.map 文件

软件方式编译 => WebStorm => filewatch、Visual Studio Code => Live Sass Compiler、koala、codekit、sublime 的插件 SASS-Build
前端自动化工作流编译 => grunt-sass、gulp-ruby-sass

语法格式：

  • .scss => 在 CSS3 语法的基础上进行拓展，所有 CSS3 语法在 SCSS 中都是通用的，同时加入 Sass 的特色功能
  • .sass => 缩进格式 sass，(1) 使用缩进方式代替花括号 {}，(2) 以换行代替分号 ;
    格式互换：
# Sass => SCSS
sass-convert style.sass style.scss

# SCSS => Sass
sass-convert style.scss style.sass

特性

变量

  • 声明方式 => $<name>: <val>;
  • 作用域（!global 声明可以将局部变量转换为全局变量）
  • 变量名中的 - 和 _ 是互通的，如：$link-color 和 $link_color 指的是同一个变量

嵌套

引入两个概念 => 父选择器，子选择器

例子：

// (1) 关系选择器相关语法：
father{ son{ ... } } => father son{ ... }
father{ > son{ ... } } => father > son{ ... }
father{ + son{ ... } } => father + son{ ... }
father{ ~ son{ ... } } => father ~ son{ ... }

// (2) & 符号 => 将父选择器作为子选择器的子串
<father>{ <part_A>&<part_B>{ ... } } => <part_A><father><part_B>{ ... }

// (3) 群组选择器相关语法1
f1, f2{ son{ ... } }
=>
f1 son{ ... }
f2 son{ ... }

// (4) 群组选择器相关语法2
father{ s1, s2{ ... } }
=>
father s1{ ... }
father s2{ ... }

// (5) 嵌套属性
selector{
	attr_prefix: { attr1: val1; attr2: val2; ... }
}
=>
selector{
	attr_prefix-attr1: val1;
	attr_prefix-attr2: val2;
	...
}

导入

@import "example_sass" => 将当前 .scss 文件与 example_sass.scss 进行拼接；编译时，会对所有 sass 语法内容进行编译

局部文件 => 专为 @import 规则导入，为多个 .scss 文件使用的 sass 文件，通常以下划线 _ 为前缀；这种文件导入时不需要加下划线 _

注释

// ... => 生成的 .css 文件中注释会被抹去
/* ... */ => 生成的 .css 文件中注释不会被抹去
/* ... */ 出现在原生 css 不允许出现的位置上 => 生成的 .css 文件中注释会被抹去

混合器

@mixin => 声明一个可插入的 css 块

// 方式一
@mixin <mixin_name>{ ... }
// 方式二
@mixin <mixin_name>($arg1, $arg2, ...){
	...
}
// 方式三：默认参数
@mixin <mixin_name>($arg1: <default_val1>, $arg2: <default_val2>, ...){
	...
}
// 注：混合器中可使用 & 运算符

@include => 插入一个 css 块

// 方式一
@include <mixin_name>
// 方式二
@include <mixin_name>(<real_arg1>, <real_arg2>, ...)
// 方式三：键值对方式填充参数
@include <mixin_name>($arg1: <real_arg1>, $arg2: <real_arg2>, ...)

混合器的例子：

@mixin link-color($default, $hover: $default, $visited: $default){
  color: $default;
  &:hover{ color: $hover }
  &:visited{ color: $visited }
}

选择器继承

@extend <example_selector> => 当前选择器继承自 <example_selector>，即当前选择器拥有 <example_selector> 的所有属性

注：sass 的继承主要用于类选择器
注2：关系选择器不能被继承
注3：继承生成的 css 体积通常比混合器生成的 css 体积更小
注4：避免对后代选择器使用继承

占位符选择器

%example => 占位符选择器，主要用于 @extend 继承，而 @example 不被编译到 .css 中
