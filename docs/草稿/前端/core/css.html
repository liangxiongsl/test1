css 语法：

<selector> {
	<property1>: <value1>;
	<property2>: <value2>;
	...	
}

引入 css 语法：

<!-- (1) 本地导入 -->
<link href="styles/example.css" rel="stylesheet" />
<!-- (2) url导入 -->
<link href="styles/example.css" rel="stylesheet" />
<!-- (3) html 内部的全局 css -->
<style>
	<selector> {
		<property1>: <value1>;
		<property2>: <value2>;
		...
	}
</style>
<!-- (4) html 内部的局部 css -->
<p style="color: red;">example text</p>
<!-- (5) @import 规则 -->
<style>
	@import "example.css"
</style>

注：浏览器会对某些元素应用默认样式
注2：基本属性 => width, height, padding, border, margin; color, background-color, text-shadow; display
注3：属性的键值对 <property>: <value>; 称为 css 声明

例子：

  • styles/style.css
p { color: red; }
  • index.html
<head>
	<link href="styles/style.css" rel="stylesheet" />
</head>
<body>
	<p>example text</p>
</body>

实践

margin => 元素与外界的边距

  • margin: <ver_hor> => 上下左右边距
  • margin: <ver> <hor> => 竖直边距，水平边距
  • margin: <top> <hor> <bottom> => 上边距，水平边距，下边距
  • margin: <top> <right> <bottom> <left> => 上、右、下、左边距

border => 元素内容与元素边框的距离，及其样式

  • border <line=style> <line-with> <color>
  • <line-style> => none, hidden, dotted, dashed, solid, double, groove, ridge, inset, outset
  • <line-width> => <length>, thin, medium, thick
    注：<line=style>, <line-with>, <color> 三者顺序无关紧要

float: left | right => 左/右 浮动

层叠 => 根据 资源顺序、优先级、重要程度 决定显示的冲突的选择器

选择器

属性选择器：

  • [attr] => 匹配存在 attr 的元素
  • [attr='val'] => 匹配 attr 属性为 'val' 的元素
  • [attr~='val'] => 若 'val' 包含于 attr.split(' ')，则匹配该元素
  • [attr|='val'] => 若 'val-' 为 attr 的前缀，则匹配该元素
  • [attr^='val'] => 若 'val' 为 attr 的前缀，则匹配该元素
  • [attr$='val'] => 若 'val' 为 attr 的后缀，则匹配该元素
  • [attr*='val'] => 若 'val' 为 attr 的字串，则匹配该元素
  • [attr='val' i] => 大小写不敏感

伪类：

  • :first-child, :last-child, :only-child
  • :blank => 输入值为空的 <input>
  • :checked => 单选/多选框 选中
  • :hover, :focus, :active => 鼠标悬停 / 元素聚焦（仅限表单等元素） / 元素激活（如：点击）
  • :link, :visited => 已访问/未访问 的链接
  • :local-link => 指向当前域的链接
  • :current()
  • :default
  • :dir() => 匹配文字书写方向，如：:dir(ltr) 或 :dir(rtl)
  • :empty => 没有儿子元素
  • :focus-visible => 聚焦且可见
  • :lang() => 根据 lang 属性匹配元素，如：:lang(en-US)
  • :first => 分页媒体的第一页，如：@page :first
  • :left / :right => 分页媒体的 奇数页/偶数页（仅有 margin/padding/border/background 等属性生效）
  • 父子关系相关
    • :first-child, :last-child => 当前元素是父元素的 第一个/最后一个 儿子
    • :first-of-type, :last-of-type => 当前元素是父元素的 第一个/最后一个 同类型儿子
    • :nth-child() => 当前元素为父元素的特定索引下的元素，如：:nth-child(2n + 3) 匹配索引为 3/5/7 的元素、:nth-child(-2n + 3) 匹配索引为 3/1 的元素
    • :nth-last-child => 先翻转父元素的子元素集合，再使用 :nth-child() 的规则
    • :nth-of-type, :nth-last-of-type => 与 :nth-child()/ :nth-last-child 类似
    • :only-child, :only-of-type => 当前元素为父元素仅有的 儿子 / 对应类型的儿子
  • 表单相关
    • :valid, :invalid, :enabled,:disabled => 合法/非法/可用/禁用 的表单元素
    • :required, :optional => 必填/非必填 表单元素
    • :out-of-range => 表单元素超出值域
    • :placeholder-shown => 显示占位文本的 <input>
  • :past()
  • :playing, :pause => 音频/视频等元素正在 播放/暂停
  • :read-only / :read-write => 只读/可读写 的元素
  • :root => 文档的根元素
  • :scope
  • :targe => 当前 url 带有 #example-id，且当前元素有 example-id 作为 id
  • :has()
  • :is(), :where() => 匹配传入的多个选择器中任意一个，如：:is(a:link, input:enabled)
  • not()
    注：有些 : 开头的是伪元素

伪元素：

  • ::first-letter, ::first-line => 匹配当前已匹配元素 首字母 / 首行
  • ::before, ::after => 匹配当前已匹配元素的实际内容 之前/之后 的可样式化元素
  • ::selection => 匹配当前已匹配元素 选中的内容
  • ::grammar-error, ::spelling-error

继承(inheritance)

优先级(speciality)

优先级一般能用三元组来度量：

  • id => id选择器，如：#id
  • class => 类/属性/伪类 选择器，如： .class, [attr], [attr=val], :fack-class
  • type => 类型/伪元素 选择器，如：el, ::fack-el
    注：组合器（如：/>/+/~/|/||/,）、通配选择器 *、伪类 :where() 不参与优先级计算
    注2：:is()/:has()/:not() 伪类本身不增加权重，但是它的参数会增加权重
    注3：内联样式具有更高的优先级，而 !important 具有最高优先级
    注4：@scope

例子：
#my-id => 1-0-0
[id="my-id"] => 0-1-0
:where(#my-id) => 0-0-0
el:not(#fakeId#fakeId#fakeID) => 3-0-1
:is(#fakeID#fakeId#fakeID, el) => 3-0-0
#myElement input.myClass => 1-1-1

创建层叠层：

  • @layer <layer1>, <layer2>, ...
  • @layer { /* css 规则... */ } 匿名层叠层，@layer <layer> { /* css 规则... */ } 具名层叠层
  • @import url(example.css) <layer> => 将 css 导入层叠层
    注：若层叠层在上文中创建，则不会重新创建
    注2：@layer "father-layer.sub-layer" => 会在 father-layer 中创建嵌套的 sub-layer
    注3：创建的层叠层都会是 root layer 的子层

层与层之间的优先级：

  • 父层 > 子层
  • 在同一层次中：较后创建的层叠层 > 较早创建的层叠层

例子：

@layer a {
	div{color: red}
}
@layer b {
	/* 优先级更高 */
	div{color: green}
}

例子2：

/* (1) */
div{color: red}
@layer a {
	/* (2) */
	div{color: green}
	@layer b {
		/* (3) */
		div{color: blue}
	}
}

/* 优先级顺序： (1) > (2) > (3) */

例子3：

@layer a {
	@layer b {
		/* 优先级更高 */
		div:not(#el){color: green}
	}
}
@layer a.b {
	div{color: red}
}

选择器

参考：伪类, 伪元素

选择器类型 描述 例子
元素/标签/类型选择器 指定 html 标签 p => <p>
id 选择器 指定具有特定 id 的标签（一个标签最多只有一个 id） #my-id => <element id="my-id">
class 选择器 指定具有特定类的元素（标签的 class 属性可以指定多个以空格分隔的类） .my-class => <element class="my-class">
属性选择器 指定拥有某些属性，或某些属性匹配某些值的标签 1. 存在：a[href] => <a href="xxx">
2. 完全匹配：a[href="https://example.com"] => <a href="https://example.com">
3. 匹配以空格分隔的子串：a[class~="my-class"] => <a class="my-class ...">
4. a[lang\|="zh"] => 匹配 lang 属性为 zh 的，或属性带有前缀 zh- 的 a 标签
5. a[class^="prefix"] => 匹配 class 属性值前缀为 prefix 的 a 标签
6. a[class$="suffix"] => 匹配 class 属性值后缀为 suffix 的 a 标签
7. a[class*="substr"] => 匹配 class 属性值包含 substr 的 a 标签
伪类选择器 指定标签的状态 a:hover => 有鼠标悬停的 <a> 标签
伪元素选择器 指定标签的某个部分 p::first-line => 选择 <p> 标签的第一行
关系选择器 使用不少于两个选择器选择标签/元素
关系选择器类型 描述 例子
后代选择器 article p => p 是article 的后代
子代选择器 article > p => p 是 article 的儿子
邻接兄弟选择器 选中紧随某个兄弟的标签 img + p => p 的上一个兄弟是 img
通用兄弟 选中具有相同父亲的兄弟标签 img ~ p => p 的一个兄弟是 img

非关系选择器 => 多个非关系选择器可以任意组合为一个选择器组合，如 a#my-id.my-class[my-attr]::before
关系选择器 => 多个选择器组合可以通过空格或+或-任意组合为一个约束更多的选择器

布局

position: enum

  • static => 默认的布局 => 不支持 top, right, left, bottom, z-index 属性
  • relative => 相对布局，计算完 static 对应的位置后，再通过 top, right, left, bottom, z-index 等调整位置 => 对 table-*-group, table-row, table-column, table-cell, table-caption 元素无效
  • absolute => 绝对布局，元素被移除正常文档流，再相对最近的非 static 祖先元素进行相对布局（并且该祖先的 transform / perspective / filter / backdrop-filter 非 none）
  • sticky => 粘性布局，相对最近滚动祖先 / 最近块级祖先 / table-related元素 进行布局
