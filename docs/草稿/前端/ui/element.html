cascader(联级选择器)

支持的功能：

  • 标签树的选择
  • 单选 / 多选
  • 懒加载
  • 搜索选择
el-cascader 的属性 用途 属性值
options: dict[] 一颗树形的数据源 每个节点至少为 {label: str, value: any, children: dict[]}
props: dict 一些配置选项
v-model: any[] | any[][] 选中一个或多个的选项的路径
clearable: bool 可清空 false
show-all-levels: bool 是否完全显示选项的路径 true
filterable: bool 是否可根据选项的 label 进行筛选 false

注：options 使用数组表示子节点，意味着它是有序树
注2：options 和 props 指定的懒加载，二选一

构造一棵随机树的例子：

let dep = 2+Math.random()*2
let dfs = (root, layer)=>{
  if (layer>dep) return
  let len = 3+Math.random()*2
  root.children = []
  for (let i = 0; i < len; i++) {
    let child = {
      label: `${root.label ? `${root.label}-` : ''}${i+1}`,
      value: `${root.value ? `${root.value}-` : ''}${i+1}`,
      children: [], disabled: Math.random()>0.5 }
    root.children.push(child)
    dfs(child,layer+1)
  }
}
// 开始构造数据
let options = {}
dfs(options, 0)
options = options.children

有几种选择方式：

选择方式 props 属性的值
单选叶子(默认) { multiple: false, checkStrictly: false }
多选叶子 { multiple: true, checkStrictly: false }
单选任意节点 { multiple: false, checkStrictly: true }
多选任意节点 { multiple: true, checkStrictly: true }

props.lazy: bool, props.lazyLoad: (node,resolve)=>void 可实现动态加载数据源
模拟异步请求加载数据的例子：

<template>
  <el-cascader :props="props"></el-cascader>
</template>
<script setup>
let id = 0
let props = {
  lazy: true,
  lazyLoad: (node, resolve)=>{
    setTimeout(()=>resolve(
        Array.from({length: 4+Math.random()*2}).map((item) => {
          return {value: ++id, label: `opt-${id}`, leaf: Math.random() > 3/id, disabled: Math.random()>0.9}
        })
    ), 30+Math.random()*400)
  }
}

</script>

注：节点是否为根节点的概率随节点数增加而衰减

也可以通过带参数的 #default 插槽定义每个选项的显示：

<el-cascader v-model="select" :props="props" filterable>
    <template #default="{node, data}">
      <el-row>
        <span>{{ data.label }}</span>
        <div style="flex-grow: 1;"/>
        <span v-if="!node.isLeaf && data.children" style="font-size: 10px">({{data.children.length}})</span>
      </el-row>
    </template>
  </el-cascader>

注：#default 的内容应为行内元素，否则会造成排版布局问题

el-popconfirm 的属性 用途
title: str 气泡框的标题
@confirm: func 确认后的回调
@cancel: func 取消后的回调
confirm-button-text: str 确认按钮的文字
cancel-button-text: str 取消按钮的文字
icon: str | Component

注：el-popconfirm 仅有 #reference 一个插槽

例子：

<el-popconfirm title="确定要删除吗" @confirm="()=>{console.log('delete something')}" @cancel="()=>{console.log('nothing to do')}">
  <template #reference>
    <el-button>删除</el-button>
  </template>
</el-popconfirm>

backtop(回到顶部)

el-backtop 的属性 用途 属性值
visibility-height: num 触发滚动的阈值 200
right: num 距离页面的右边距 40
bottom: num 距离页面的下边距 40

例子：

<template>
	<el-backtop :right="rt" :bottom="bt" />
</template>
<script setup>
let rt = computed(()=>{
  return window.innerWidth * 0.05
})
let bt = computed(()=>{
  return window.innerHeight * 0.8
})
</script>

编码风格

  • 测试主界面布局或其他标签布局时，主背景 el-container 的颜色应设置为非白色；el-container 的每个儿子标签应设置为 非主背景色/非白色，并且为这些儿子设置 border-radius 是他们变成椭圆
  • 一个 element 组件容器应该定义一个 reactive<dict> 实例来存储相应的配置数据，并且该组件容器的属性名应尽量与这个实例的字段名相同 => 例子：
<template>
	<el-table :data="table.data"></el-table>
</template>
<script setup lang="ts">
import {reactive} from 'vue'
// data_base 字段用于表示原始数据的数组
// data 字段则通过 分页大小 和 当前分页位置，计算出当前的 data_base 的子数组 => data 字段使用 computed 来完成这种依赖关系（若使用事件处理则非常麻烦）
let table = reactive({data_base: [], data: []})
</script>
  • 具有依赖关系的变量之间，应优先使用 computed() 来建立依赖，不推荐使用事件驱动（后者容易增加代码混乱度）
