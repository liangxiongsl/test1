核心

cdn 方式使用 vue router 的例子：（提供了两条 url 到 vue 组件的映射）

<script src="https://unpkg.com/vue@3"></script>
<script src="https://unpkg.com/vue-router@4"></script>  
<script src="https://unpkg.com/element-plus@2.6.2"></script>  
<link rel="stylesheet" href="https://unpkg.com/element-plus@2.6.2/dist/index.css">

<div id="app">
    <h1>Hello App!</h1>
    <router-link to="/">Go to Home</router-link> <br>
    <router-link to="/about">Go to About</router-link>
    <router-view></router-view>
</div>

<script>
    const {createApp} = Vue
    const {createRouter, createWebHashHistory} = VueRouter
    
    const Home = { template: '<div>Home</div>' }
    const About = { template: '<div>About</div>' }

    // 路由规则
    const routes = [
        { path: '/', component: Home },
        { path: '/about/', component: About },
    ]

    // 路由器
    const router = createRouter({
        history: createWebHashHistory(),
        routes, // `routes: routes` 的缩写
        sensitive: false,	// => 大小写是否敏感，默认不敏感
        strict: false,		// => 是否严格匹配尾部带有 / url，默认不严格
    })
    
    const app = createApp({})
    app.use(router)  
	app.use(ElementPlus)
    app.mount('#app')
</script>

<router-link> => 以不刷新页面的方式更改 url
<router-view> => 当前显示的组件

动态路由

参考：路由匹配语法

路由参数 => 配置每条路由规则时可以使用 :param 的方式定义
=> 如 { path: 'repository/:user/:repo', component: Repository }

this.$router => 路由器（最开始是由 createRouter() 创建的对象）
this.$route => 当前的路由
this.$route.params => 获取 path 内的路径参数（路由参数）
this.$route.query => 获取查询字符串对应的对象数组

响应路由变化

方式一：（导航守卫 => 只侦听某个组件的路径参数变化）
async beforeRouteUpdate(to, from){} 选项 => 侦听并在路由参数变化前进行调用

例子：（在 Repo 组件中声明 beforeRouteUpdate 选项）

const Repo = { template: `<div>
User => {{ $route.params.user }} <br>
repo => {{ $route.params.repo }} <br>
params => {{this.$route.params}} <br>
querys => {{this.$route.query}}
</div>`,
    async beforeRouteUpdate(now, prev){
        console.log(prev.params, prev.query, now.params, now.query)
        alert()
    }
}

方式二：（侦听所有路径参数的变化）

例子：（在根组件的 created(){} 选项中添加代码）

this.$watch(
    () => this.$route.params,
    (nowParams, prevParams) => {
        console.log(prevParams, nowParams)
        alert()
    }
)

路由匹配语法

外链：Vue 路由器路径解析器 (esm.dev)

路径参数可以通过正则表达式定义 => :<arg>(<regex>)

  • :<arg> 等价于 :<arg>([^/]+)
  • :<arg>(<regex>)+ => 如：/:passage(\\d+)+ 匹配 /12，/12/23 等
  • :<arg>(<regex>)* => 如：/:passage(\\d+)+ 匹配 /，/12，/12/23 等
  • :<arg>(<regex>)? => 如：/:passage(\\d+)? 匹配 /，/12 等
    注：\ 需要转义

<router>.resolve(<rule>) 解析路由：

例子：

  • 假设有一条路由规则 =>
{ name: 'example_route', path: '/:param1(\\d*)+/:param2([a-z]*)', component: example_component }
  • 解析一条路由 =>
const route = router.resolve({name: 'example_route', params: { param1: ['123','456'], param2: 'liangxiongsl' }})
console.log(route)
console.log(route.href) // => #/123/456/liangxiongsl

createRouter(options: Record<string, unknown>) 的配置选项：

  • sensitive => 大小写是否敏感，默认为 false
  • strict => 是否严格匹配尾部带有 / url，默认为 false

嵌套路由

路由可以配置多个，而且还支持嵌套，于是路由又可以认为是路由森林

大致工作方式 => 对于一条 url，若匹配到了路由森林的一条路径，那么 vue 会一路渲染路径上经过的路由节点对应的组件
（父路由组件使用 <router-view> 插入子路由组件）

每一条路由都满足如下规则：

路由参数 用途 补充
name: string 路由规则名 可以用于路由解析 <router>.resolve(<rule>)
path: string path 的匹配字符串（路由规则） 若首个字符不为 /，那么会为其添加父路由的 path/ 为前缀
component: object 路由对应的组件 模板中提供的 <router-view> 用于渲染一个子路由的组件
children: object[] 子路由

一个路由配置的例子：

const route_forest = [
    {
        name: 'user',
        path: '/:uid',
        component: {
            template: `
        <h2>[layer 2] welcome {{$route.params.uid}} !!!</h2>
        <button><router-link :to="this.PATH + ''">home</router-link></button>
        <button><router-link :to="this.PATH + '/avatar'">avatar</router-link></button>
        <button><router-link :to="this.PATH + '/profile'">profile</router-link></button>
        <br>
        <router-view></router-view>`,
            data(){
                return{ PATH: this.$router.resolve({name: `user`, params:{uid: this.$route.params.uid } }).href.substring(1) }
            },
        },
        children: [
            {path: '', component: {template: '<h3>[layer 3]home place</h3>'}},
            {path: 'avatar', component: {template: '[layer 3]<br><img src="https://t.mwm.moe/mp">'}},
            {path: 'profile', component: {template: '<h3>[layer 3]{{$route.params.uid}} is awesome</h3>'}},
            {path: ':other(.*)', component: {template: '<h3>[layer 3]what?</h3>'}},
        ]
    },
    {
        path: '/:other(.*)',
        component: {
            template: `
        <h1 style="color: red">404 Not Founded !!!</h1>
        "{{this.$route.path}}" is not a regular path`
        }
    }
]

路由的作用
=> 将组件组合成一棵路由组件树
=> 路由器将 url 映射 / 导航 到路由组件树中的一条组件路径上，最后对该路径进行渲染

编程式导航

外链：历史记录 API - Web API 接口 |MDN网络 (mozilla.org)

<router>（this.$router 或 useRouter()）提供的导航方法：

方法 用途 等价方式
push(path_config: string | object) 导航到指定路径 <router-link to="...">
replace(path_config: string | object) 替换导航到指定路径 <router-link to="..." replace>
go(diff: integer) 相对导航 window.history.go(<diff>)
forward() 前进
back() 回退
普通导航

<router>.push(path_config: string | object) => 导航到指定的路径中，功能类似于 <router-link :to="...">
注：<router> => 指的是 this.$router 或 useRouter()

配置项 解释
path: string 或 {path: string} 要导航的路径
{path: string, query: Record<string, unkown>} 带查询参数的导航
{path: string, hash: string}
{name: string, params: Record<string, unkown>} 解析某一条路由并导航
替换式导航

<router>.replace(path_config: string | object) => 导航到指定的路径中，并且不会添加历史记录，功能类似于 <router-link :to="..." replace>

相对导航

<router>.go(diff: integer) => 导航前进 <diff> 步（可负），类似于 window.history.go(<diff>)
<router>.forward() => 导航前进 1 步
<router>.back() => 导航后退 1 步

命名路由

命名视图

一个路由也可以对应多个组件，例如：
=> 模板

<template>
    <router-view name="left"></router-view>
    <router-view></router-view>
    <router-view name="right"></router-view>
</template>

=> 某个一级路由：

// 先定义组件 => Home, LeftSidebar, RightSidebar
{
  path: '/',
  components: {
    default: Home,
    left: LeftSidebar,
    right: RightSidebar,
}

注：router-view 的 name 默认值为 default

重定向和别名

redirect => 指定要重定向 path 或 另一条路由规则，还支持调用回调方法（逻辑上类似于 <router>.push()）
alias => 指定路由规则的别名，匹配的路由别名会替换为 path 字段指定的路由规则（逻辑上类似于 <router>.replace()）

路由规则 用途 补充
redirect: string 指定重定向的 path 不以 / 开头时可以重定向到相对位置
redirect: {name: string} 指定要重定向的另一条路由规则
redirect: (to: RouteLocation)=> string | {name: string} 重定向前预处理 to 为重定向前的路由
alias: string | string[] 指定路由规则的别名 可以使用父路由的参数 :<param>

例1：{path: '/:uid/(.*)', redirect: 'profile'} 会将 /liangxiongsl/djiad 重定向到 /liangxiongsl/profile（存疑）
例2：

const routes={
    path: '/user/:uid', 
    children: [ {path: 'profile', alias: ['/:id', '']} ]
}

=> 会将 /liangxiongsl 或 /user/liangxiongsl 替换为 /user/liangxiongsl/profile

路由组件传参

路由规则 用途
props: boolean 指定是否为路由的 default 子组件的 props 选项传递路由参数
props: Record<string, boolean> 指定是否为路由的某些子组件的 props 选项传递路由参数
props: object 将对象传递给子组件的 props 选项
props: (route: RouteRecordRaw) => Record<string, any> 根据当前的路由记录动态传参给子组件的 props 选项

其他传参方式 => 将 props 传递给路由组件 | Vue Router (vuejs.org)

部署

不同的历史模式 | Vue Router (vuejs.org)

createRouter(<options>) 提供了一条 history 配置
=> hash 模式：createWebHashHistory() => 内部传递 url 时使用了 # 拼接 path 的方式更新页面 => 适合 .html 应用
注：url 会形如 https://example.com/index.html#user/1506218507
=> memory 模式：createMemoryHistory() => 不会假定自己处理浏览器环境，不会有历史记录 => 适合 node 环境 或 SSR
=> html5 模式： createWebHistory() => 使用正常的 url 更新页面
注：url 形如 https://example.com/user/1506218507
