开发方式

根据 文件格式 和 api 方式的不同，vue 组件的开发方法有 4 种（存疑）
（疑问：html 方式是否存在 组合式 api？）

文件格式：
=> .html 方式

  • cdn 导入 vue3 => <script src="//cdn.jsdelivr.net/npm/vue@3"></script> => 会提供对象 Vue
  • 书写组件代码 => <script> Vue.createApp(<options>).mount(<selector>) </script>
    => .vue 方式
  • 选择性安装一些包 => npm install vue3 element-plus @element-plus/icons-vue, npm install -D vite @vitejs/plugin-vue typescript vue-tsc
  • 在 .vue 中的 <template>, <script>, <style> 标签书写组件代码（其中 <script> 默认导出一个选项对象 { ... } 或者使用组合式的方式书写）
  • .js 或 .ts 通过 import 导入 .vue => 通过 createApp(<options>).mount(<selector>) 挂载
  • .html 通过 <script type="module" src="xxx/yyy.ts"></script> 导入 .js 或 .ts；声明组件要挂载的容器 <div id="<selector>"></div>

api 方式：
=> 选项式

  • 在 <script> 中默认导出一个选项对象，其定义了当前组件的主要逻辑
    => 组合式
  • 在 <script setup> 中书写组件逻辑，以导入模块的方式表示组件的逻辑

插件

参考：App => use()

第三方的 vue 组件库包含大量 vue 组件供开发使用

可以用 <App>.use(<plugin>) 的方式全局引入组件库，也可以单独 import 该组件库的组件并在选项 component 中注册特定组件

以下两个全局引入组件库的例子：

以 element-plus 为例（element-plus 是一个 组件库，也可以说是一个 插件）
=> 安装：npm install --save element-plus @element-plus/icons-vue
=> 在 .ts中添加代码：

import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'
import "element-plus/theme-chalk/src/message.scss"; // => 支持 ElMessage
app.use(ElementPlus);

又如 naive-ui：
=> 安装： npm i -D naive-ui, npm i -D vfonts
=> 在 .ts 中添加代码：

import Naive from 'naive-ui'  
app.use(Naive)

子传父事件

  • 子组件
    => 通过 emits 选项声明可供发送的事件
    => 通过 $emit('<event>') 发送事件给父组件

父传子标签——插槽

  • 子组件 => 在任意多处插入 <slot> 插槽标签
  • 父组件 => 在子组件标签的 innerHTML 中定义要传给子组件的 html 内容

根传孙数据：

  • 孙组件 => 通过 inject 选项注入响应式状态
  • app => <App>.provide()

deactivated 若组件实例是 <KeepAlive> 缓存树的一部分，当组件从 DOM 中被移除时调用

组合式api

script 标签添加 setup 属性则可以正式使用组合式 api，比如：

<-- 使用 javaScript -->
<script lang="js" setup> ...  </script>
<-- 使用 typescript -->
<script lang="ts" setup> ...  </script>

涉及的模块：

  • @vue.reactivity => ref()
  • @vue.runtime-core => computed()

ref<T>(value: T): Ref<UnwrapRef<T>> 或 ref<T = any>(): Ref<T | undefined> => 创建响应式状态

  • Ref 的 value 属性可以得到它的值（template 模板标签内不用调用 value ）

computed<T>(getter: ComputedGetter<T>, debugOptions?: DebuggerOptions): ComputedRef<T>
或 computed<T>(options: WritableComputedOptions<T>, debugOptions?: DebuggerOptions): WritableComputedRef<T>
=> 创建计算方法

箭头函数或是普通函数都可以作为响应式方法 => 例如：

const method1 = () => { console.log("hello world") }
function method2(){
	console.log("hello world")
}

provide/inject

provide(key: str, val: any) => 祖先组件向子孙组件提供一个只读的对象
inject(key: str): any => 子孙组件注入一个祖先组件的对象

注：provide() 方法只是提供一个祖先组件向子孙组件单向传输数据的方式

祖孙双向通信

原理很简单 => 用 provide() 向子组件提供 对象的 setter 方法即可

例子：=> 

// tool.js
import {ref} from 'vue'
export class Lx_Context{
    constructor(val) {
        this._ = val
    }
    set value(val){
        this._ = val
    }
    get value(){
        return this._
    }
}
export function construct(val){
    return ref(new Lx_Context(val))
}

// main.js
import {construct} from './components/utils/tool.js'
app.provide('key', construct('val'))
  • 子组件 subcomponent.vue 中使用 key 对应的对象：
<script setup>
	import {inject} from 'vue'
	let inject_obj = inject('key')
	console.log(inject_obj.value.value) # => 'val'
</script>
<template>
	{{inject_obj.value}} 	<!-- => 'val'   -->
</template>

watchEffect()

立即运行一个函数，同时响应式地追踪其依赖，并在依赖更改时重新执行

watchEffect(
	effect: ()=>void, 
	opts?: {flush: 'pre' | post | 'sync'}
): WatchStopHandle

例子：http 查询参数更改时触发 api 请求

import {get} from '@/components/utils/http'
import {watchEffect,ref} from 'vue'
import {useRoute} from 'vue-router'
let route = useRoute()

let data = ref()

let stopHandle = watchEffect(async ()=>{
  await get(`User/${route.query.id ? route.query.id : 33}/d/`).then(res => data.value=res)
})
// 不需要这种副作用时，应该调用 stopHandle()

注：在组合式 api 下直接使用 await get() 等请求会导致 vue 运行出错

编辑器：（ctrl + d 删除一行文本）

<script>
let input = ref('')
onMounted(()=>{
  let textarea = document.getElementsByTagName('textarea')[0] as HTMLTextAreaElement
  textarea.addEventListener('keydown',(e)=>{
    if (e.key==='d'&&e.ctrlKey){
      let st = textarea.selectionStart, ed = textarea.selectionEnd
      st = input.value.substring(0, st).lastIndexOf('\n')
      ed = ed + input.value.substr(ed).indexOf('\n')
      if (st===-1) st = 0
      if (ed===-1) ed = input.value.length
      textarea.setRangeText('', st+1, ed+1)

      e.preventDefault()
    }
  })
})
</script>
<template>
	<textarea v-model="input"></textarea>	
</template>
