开发方式

api风格

参考：vue | api-styles

Vue3 的组件可以按两种不同的风格书写

  • 选项式 API
  • 组合式 API
选项 解释
data(){ return { <key1>: <val1>, ... } } 其放回的对象的键值对会暴露在 this 上
methods: { <function>(){ ... }, ... } 定义一些用来更改状态或触发更新的函数
mounted(){ ... } 一种生命周期钩子——当组件挂载完成后调用
选项式 api
<script>
export default {
  // data() 返回的属性将会成为响应式的状态
  // 并且暴露在 `this` 上
  data() {
    return {
      count: 0
    }
  },

  // methods 是一些用来更改状态与触发更新的函数
  // 它们可以在模板中作为事件处理器绑定
  methods: {
    increment() {
      this.count++
    }
  },

  // 生命周期钩子会在组件生命周期的各个不同阶段被调用
  // 例如这个函数就会在组件挂载完成后被调用
  mounted() {
    console.log(`The initial count is ${this.count}.`)
  }
}
</script>

<template>
  <button @click="increment">Count is: {{ count }}</button>
</template>
组合式 api
<script setup>
import { ref, onMounted } from 'vue'

// 响应式状态
const count = ref(0)

// 用来修改状态、触发更新的函数
function increment() {
  count.value++
}

// 生命周期钩子
onMounted(() => {
  console.log(`The initial count is ${count.value}.`)
})
</script>

<template>
  <button @click="increment">Count is: {{ count }}</button>
</template>

html式组件 / 单文件组件

这两种不同的书写方式，分别对应 .html 或 .vue 文件

html式组件

注：html式组件总是绑定某个类

createApp(<options>).mount(<selector>)

  • 选项式 html 的例子：
<script type="module">
import { createApp } from 'vue'

createApp({
  data() {
    return {
      message: 'Hello World!'
    }
  }
}).mount('#app')
</script>

<div id="app">
  <h1>{{ message }}</h1>
</div>
  • 组合式 html 的例子：
<script type="module">
import { createApp, ref } from 'vue'

createApp({
  setup() {
    const message = ref('Hello World!')

    return {
      message
    }
  }
}).mount('#app')
</script>

<div id="app">
  <h1>{{ message }}</h1>
</div>
单文件组件

参考：vue | sfc

Vue 的单文件组件是网页开发中 HTML、CSS 和 JavaScript 三种语言经典组合的自然延伸。<template>、<script> 和 <style> 三个块在同一个文件中封装、组合了组件的视图、逻辑和样式。完整的语法定义可以查阅 SFC 语法说明

  • 选项式 单文件 的例子：
<script>

  data() {
    return {
      message: 'Hello World!'
    }
  }
}
</script>

<template>
  <h1>{{ message }}</h1>
</template>
  • 组合式 单文件 的例子：
<script setup>
import { ref } from 'vue'

const message = ref('Hello World!')
</script>

<template>
  <h1>{{ message }}</h1>
</template>

挂载应用

应用实例必须挂载某个容器后才会渲染出来

  • 挂载 => <app>.mount('<css选择器>')，会返回 根组件实例；如：app.mount('#app')
  • 容器 => 如，<div id="app"></div>

响应式

响应式状态

组件的响应式状态通过方法选项 data(){ ...; return {<key>:<val>, ...} } 来定义
=> <component>.<key> 可以引用某个键值；其他方法选项或钩子选项也可以通过 this.<key> 来引用键值

例子：

<script>
export default {
  data() {
    return {
      count: 1
    }
  },

  // `mounted` 是生命周期钩子，之后我们会讲到
  mounted() {
    // `this` 指向当前组件实例
    console.log(this.count) // => 1

    // 数据属性也可以被更改
    this.count = 2
  }
}
</script>

注：data 上定义的属性尽量不要包含前缀 $ 或 _
注2：在 Vue 3 中，数据是基于 JavaScript Proxy（代理） 实现响应式的。使用过 Vue 2 的用户可能需要注意下面这样的边界情况：

<script>
export default {
  data() {
    return {
      someObject: {}
    }
  },
  mounted() {
    const newObject = {}
    this.someObject = newObject

    console.log(newObject === this.someObject) // false
  }
}
</script>

响应式方法

响应式方法应使用键值对选项 method: { <function>(){ ... }, ... } 来定义
响应式方法可以在三种情况下访问：

  1. <component>.<method>
  2. 方法选项或钩子选项内的 this.<method>
  3. 模板内

例子：

<script>
export default {
  data() {
    return {
      count: 0
    }
  },
  methods: {
    increment() {
      this.count++
    }
  },
  mounted() {
    // 在其他方法或是生命周期中也可以调用方法
    this.increment()
  }
}
</script>

要等待 DOM 更新完成后再执行额外的代码，可以使用 nextTick() 全局 API：

<script>
import { nextTick } from 'vue'

export default {
  methods: {
    async increment() {
      this.count++
      await nextTick()
      // 现在 DOM 已经更新了
    }
  }
}
</script>

模板语法

在绝大多数情况下，Vue 推荐使用模板语法来创建应用。然而在某些使用场景下，我们需要使用渲染函数

模板语法可以实现 动态文本、动态 html、动态属性、动态指令参数等

文本绑定

模板中 {{ <value> }}
（value可以是响应式状态）

例子：

<template>
{{ message }}
</template>
<script>
export default{
	data(){
		message: "hello"
	}
}
</script>

渲染式绑定

模板中标签属性 v-html（或者称为 vue 指令）可以将某个响应式状态绑定到该标签的 innerHTML

例子：

<template>
<span v-html="rawHtml"></span>
</template>
<script>
export default{
	data(){
		rawHtml: "<span style="color: red">This should be red.</span>"
	}
}
</script>

注：在网站上动态渲染任意 HTML 是非常危险的，因为这非常容易造成 XSS 漏洞。请仅在内容安全可信时再使用 v-html，并且永远不要使用用户提供的 HTML 内容

属性绑定

绑定单个属性

模板中标签属性 v-bind:<attribute> 可以将某个响应式状态绑定到该标签的某个属性

该指令也可以简写为 :<attribute>

注：<div :id></div> 等价于 <div v-bind:id></div>
注2：布尔型属性的绑定

绑定多个属性

模板中标签属性 v-bind 可以将某个 map 类型的响应式状态绑定到多个属性

例子：

<template>
<div v-bind="map"></div>
</template>

<script>
data() {
  return {
    map: {
      id: 'container',
      class: 'wrapper'
    }
  }
}
</script>

  • 参数——某些指令会需要一个参数；例如 v-bind:href="url"，其中 href 就是参数

计算属性

只读计算属性

只读计算属性通过 computed: { <function>(){...}, ... } 来定义

注：

  • 计算属性与响应式状态的定义类似，但前者会将函数的返回值进行缓存，加快第二次数据访问，从而优化性能
  • 计算属性的使用方法与响应式状态类似（不需要加括号）
  • 当响应式依赖更新时，计算属性的缓存值也随之更新

可读写计算属性

可读写计算属性通过 computed: { <attr>: { get(){ return ...; }, set(<val>){ ... } }, ... } 来定义

  • 当运行 this.<attr> 时，会调用其 getter 方法
  • 当运行 this.<attr> = ... 时，会调用 setter 方法

例如：

<script>
export default {
  data() {
    return {
      firstName: 'John',
      lastName: 'Doe'
    }
  },
  computed: {
    fullName: {
      // getter
      get() { // this.fullName => 等于该方法的返回值
        return this.firstName + ' ' + this.lastName
      },
      // setter
      set(newValue) { // this.fullName = "liang xiong"  =>  会调用该方法
        // 注意：我们这里使用的是解构赋值语法
        [this.firstName, this.lastName] = newValue.split(' ')
      }
    }
  }
}
</script>

也就是说，可读写的计算属性建立在响应式状态之上，而用法上与响应式状态几乎相同

迭代渲染

  • v-for="(item, index) in <array>"
  • v-for="(value, key, index) in <object>"

类似于 js 代码：

<array>.forEach((item, index) => {
	// ...
})
<object>.forEach((value, key, index) => {
	// ...

:key => 用于给每个项目提供唯一标识，以提供dom元素渲染的动态更新

其他：

  • 若数组使用了 push(),pop(), shift(), unshift(), sort(), reverse(), splice() 等变更方法来更新自己，Vue 会对相关更新进行渲染
  • 若数组使用了 filter(), concat(), slice() 等非变更方法来更新自己，Vue 也会高效地重新渲染（未作改变的item会重用）

事件处理

两种事件处理器：
(1) 内联事件处理器——处理器为 js 代码，可以配合事件变量 $event 来使用，也可以使用lambda表达式；如：@click="greet($event)" 或 @click="(event) => ..."
(2) 方法事件处理器——如：@click="greet"
（注：假设 greet 是一个 methods 中定义的响应式方法，其定义形如 greet(event){ ... }；其中 event.target.tagName 可以获取触发事件的标签名）

方法与内联事件判断：

  • foo、foo.bar 和 foo['bar'] 会被视为方法事件处理器
  • foo() 和 count++ 会被视为内联事件处理器

基本的事件修饰符：

  • .stop => 内部事件处理后，停止传递给外部事件 => 本质是调用了 event.stopPropagation()
  • .self => 不处理来自于内部的事件，但提供外部事件的传递
  • .capture => 使用捕获模式 => 在内部标签的事件被其处理前，先行处理该事件（一条事件处理链上有多个 .capture，那么最外层的最优先）
  • .prevent => 事件触发后不再重新加载页面，如 <form @submit.prevent="onSubmit"></form> => 本质是调用了 event.preventDefault()（注：表单提交后默认会刷新或跳转）
  • .once => 事件最多触发一次
  • .passive => 事件的默认行为将立即发生而非等待事件处理函数完成调用
    注：修饰符可以支持链式书写，如 @click.stop.prevent
    注2：若修饰符的调用顺序不同，效果也可能不一样

<div @click="console.log('外')">外部
  <div @click.stop="console.log('中')">中部

表单输入绑定

基本使用

v-model 用于绑定输入标签的输入值，它会根据所使用的元素自动使用对应的 DOM 属性和事件组合

  • 文本类型的 <input> 和 <textarea> 标签会绑定 value 属性并侦听 input 事件
  • <input type="checkbox"> 和 <input type="radio"> 会绑定 checked 属性并侦听 change 事件
  • <select> 会绑定 value 属性并侦听 change 事件

文本框/文本区域的例子：

<input :value="text" @input="event => text = event.target.value">
<!--等价于-->
<input v-model="text">

复选框的 3 个例子：

<!-- ok 是一个 bool 变量，复选框的选中状态与 ok 的值绑定 -->
<input type="checkbox" v-model="ok" id="_id_"/>
<label for="_id_">{{ ok }}</label>

<!-- toggle 是一个字符串，选中时 toggle 为 true-value 的值，反之为 false-value 的值 -->
<input type="checkbox" v-model="toggle" id="_id_" true-value="yes" false-value="no"/>
<label for="_id_">{{ ok }}</label>

<!-- res 为一个数组，若某个复选框被选中，那么 value 属性将添加进 res -->
<div v-for="x in raw">
  <input type="checkbox" :value="x" v-model="res" :id="x">
  <label :for="x" >__{{x}}__</label>
</div>
{{res}}

单选框的例子：

<!-- selected 为一个 bool 变量，若某个单选框被选中，那么 value 属性将赋值给 selected -->
<div v-for="x in raw">
  <input type="radio" :value="x" v-model="selected" :id="x">
  <label :for="x" >__{{x}}__</label>
</div>
{{selected}}

选择器的 2 个例子：（其功能类似于单选框）

<!-- selected 为一个 bool 变量，若某个选项被选中，那么 value 属性将赋值给 selected -->
<select v-model="selected">
  <option disabled value="">please select one</option>
  <option v-for="x in raw" :value="x">__{{x}}__</option>
</select>

<select v-model="selected" multiple>
  <option v-for="x in raw" :value="x">__{{x}}__</option>
</select>

注：理论上 复选框/单选框/选择器 的绑定值可以是任意类型

修饰符

v-model 的修饰符：

  • .lazy => v-model 在每次 change 事件后更新数据；否则，默认情况下会在 input 事件后更新数据
  • .number => 将输入转换为数字
  • .trim => 去除输入内容的两端的空格

组件上使用 v-model

组件 v-model | Vue.js (vuejs.org)

假设 myInput 是一个子组件，那么 <myInput v-model="val"/> 将被 Vue 解释为 => 也就是说，会给子组件标签声明一个 modelValue 属性，并且会侦听子组件传入的 update:modelValue 事件 

<myInput :model-value="val" @update:model-value="newValue => val = newValue">

相应的子组件 myInput 可以是：

<template>
  <input :value="modelValue" @input="$emit('update:modelValue', $event.target.value)">
</template>

<script>
export default {
  name: "myInput",
  props:['modelValue'],
  emits:['update:modelValue']
}
</script>

也可以用计算属性：

<template>
  <input v-model="value">
</template>

<script>
export default {
  props:['modelValue'],
  emits:['update:modelValue'],
  computed:{
    value: {
      get(){ return this.modelValue },
      set(val){ this.$emit('update:modelValue', val) }
    }
  }
}
</script>
组件 v-model 的参数

组件 v-model 的参数—v-model:<param> 会给子组件标签声明一个 <param> 属性，并且会侦听子组件传入的 update:<param> 事件

侦听器

浅层侦听

组件选项 watch 通过为响应式状态声明侦听器函数来侦听该响应式状态的变化，如：watch: { <state>(newValue, oldValue){}, ... }

=> watch 默认是浅层的：被侦听的响应式状态，仅在被赋新值时，才会触发回调函数——而嵌套属性的变化不会触发

深层侦听、即时回调、回调的触发时机

watch: {
  someObject: {
    handler(newValue, oldValue) {
      // 注意：在嵌套的变更中，
      // 只要没有替换对象本身，
      // 那么这里的 `newValue` 和 `oldValue` 相同
    },
    deep: true,			// => 深层侦听 
    immediate: true,	// => 创立侦听器时，立即执行
    flush: 'post'		// => 侦听器会在 DOM 更新后回调；而默认会在 DOM 更新前回调
  }
}

命令式创建侦听器

<script>
export default {
  created() {
    this.$watch('attr', (newAttr) => {
      // ...
    })
  }
}
</script>

注：this.$watch() 返回一个函数，调用该函数可以停止侦听

模板引用

为标签申明 ref 属性，组件挂载（mounted）后，可以通过 this.$refs.<ref> 引用该标签（this 可以指根组件，或者可以在生命周期构造上引用）

函数模板引用

ref 属性可以绑定为一个函数，会在每次组件更新时都被调用，该函数会收到元素引用作为其第一个参数：

<input :ref="(el) => { /* 将 el 赋值给一个数据属性或 ref 变量 */ }">

注：这里需要使用动态的 :ref 绑定才能够传入一个函数。当绑定的元素被卸载时，函数也会被调用一次，此时的 el 参数会是 null。你当然也可以绑定一个组件方法而不是内联函数

对子组件标签的引用

如果一个子组件使用的是选项式 API ，被引用的组件实例和该子组件的 this 完全一致，这意味着父组件对子组件的每一个属性和方法都有完全的访问权
