git database

tree

POST /repos/{owner}/{repo}/git/trees

  • req：
{
	tree: {
		// 文件路径
		path: str,
		// 文件模式：文件(blob), 可执行文件(blob), 目录(tree), 子模块(commit), 指定符号链接路径的 blob
		mode: '100644' | '100755' | '040000' | '160000' | '120000',
		type?: 'blob' | 'tree' | 'commit',
		// 若为 null，则删除文件
		sha?: sha_str | null,
		// 注：sha 与 content 只能选其一
		content?: str
	}[],
	base_tree: sha_str
}
  • res：{sha: sha_str, url: str, tree: {path: str, mode: str, type: str, sha: sha_str, size: num, url: str}, truncated: bool}

GET /repos/{owner}/{repo}/git/trees/{tree_sha}

  • res：{sha: sha_str, url: str, tree: {path: str, mode: str, type: str, sha: sha_str, size: num, url: str}, truncated: bool}

blob

POST /repos/{owner}/{repo}/git/blobs

  • req：{ content: str, encoding: 'utf-8' | 'base64' }
  • res：{ url: str, sha: sha_str }

GET /repos/{owner}/{repo}/git/blobs/{file_sha}

  • res：{ url: str, sha: sha_str, content: str, encoding: str, size: num, node_id: str }

commit

POST /repos/{owner}/{repo}/git/commits

  • req：
{
	message: str,
	tree: sha_str,
	parents?: str[],
	author?: { name: str, email: str, data?: 'YYYY-MM-DDTHH:MM:SSZ' },
	committer?: { name: str, email: str, data?: 'YYYY-MM-DDTHH:MM:SSZ' },
	signature?: str
}

REST

REST api => method, headers, media types, authentication, path/query/body parameters

method
=> GET => 检索资源
=> POST => 创建资源
=> PATCH => 更新资源
=> PUT => 替换资源/资源集合
=> DELETE => 删除资源

headers

  • Accept => application/vnd.github+json 或 application/json
  • X-GitHub-Api-Version => 指定要用于请求的 REST API 版本，参考 API Versions - GitHub Docs (kkgithub.com)
  • User-Agent => `Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36 Edg/120.0.0.0

media types => 由 Accept 标头指定

parameters => 路径参数、查询参数、正文参数

注：GET 方法不允许添加 body parameters

认证

github 基本上有两种 token：Fine-grained personal access tokens，Personal access tokens (classic)

Fine-grained 的特点：

  • 可选定最多不超过 1 年的过期时间
  • 可指定仓库
  • 可选定 仓库/用户 的读写权限

classic 的特点：

  • 可设置无过期时间上限
  • 可选定 仓库/用户 的读写权限

使用 REST API 和 JavaScript 编写脚本 - GitHub 文档 (kkgithub.com)

安装库 => npm i octokit

初始化：

import { Octokit } from "octokit";
const octokit = new Octokit({
    auth: `example_token`
})

get 请求例子：

await octokit.request("GET /repos/{owner}/{repo}/issues", {
    owner: "liangxiongsl",
    repo: "image-repo",
    per_page: 2
})

post 请求例子：

await octokit.request("POST /markdown/raw", {
    text: "Hello **world**",
    headers: {
        "content-type": "text/plain",
        "x-github-api-version": "2022-11-28",
    },
});
get 请求

获取文件 =>

  • https://github.com/<user>/<repo>/blob/<branch>/<file-path>?raw=true
  • 或 https://raw.githubusercontent.com/<user>/<repo>/<branch>/<file-path>
    例子：https://github.com/liangxiongsl/image-repo/blob/main/readme.md?raw=true
请求 用途
<user>/<repo>/blob/<branch>/<file-path>?raw=true 获取文件
