不同角色需要不同的命令

  • 独立开发者
  • 多人开发
  • 集成者
  • 存储库管理员

独立开发者：

  • git init => 初始化存储库
  • git switch / git branch => 切换分支
  • git add => 管理索引文件？（stage）
  • git diff / git status => 查看/比对所有修改
  • git commit => 推送当前分支
  • git restore => 撤销更改（与 git add 相反）
  • git checkout => 回溯到之前的提交节点（要保留往后的提交，则需要创建新的分支） / 切换分支
  • git log => 查看所有提交
  • git merge => 合并本地分支
  • git rebase => 维护 topic 分支？
  • git tag => 标记已知点？
    多人开发（或提供个人远程同步）：
  • git clone => 从 upstream 来准备本地仓库
  • git pull / git fetch => 与 upstream 保持同步
  • git push => 推送代码到 shared repository
  • git format-patch => 准备电子邮件提交
  • git send-email => 发送电子邮件提交，而无需损坏你的 MUA（邮件用户代理）
  • git request-pull => 创建更改摘要，让你的 upstream 来 pull
    集成者：
  • git am => 应用从贡献者通过邮件发送来的补丁
  • git pull => 从你信任的助手那里合并
  • git push => 发布前沿技术
  • git format-patch => 向贡献者准备和发送建议的替代方案？
  • git revert => 撤销拙略的更改
  • git reset =>
    存储库管理员：
  • git daemon => 允许匿名下载存储库
  • git shell => 使得共享中央存储库的用户以受限方式登录 shell
  • git http-backend => 提供 Git-over-HTTP 的服务端实现，以允许 fetch 和 push 服务
  • gitweb => 为存储库提供一个 web 前端，库使用 git instaweb 脚本进行设置

初始化

配置全局 user.name 和 user.email：

git config --global user.name <username>
git config --global user.email <email>

设置代理（如：使用 clash）

git config --global http.proxy 127.0.0.1:7890
git config --global https.proxy 127.0.0.1:7890

配置行结束符的首选项：（但可能造成某些命令报错）

# unix/linux 用户
git config --global core.autocrlf input
git config --global core.safecrlf true
# windows 用户
git config --global core.autocrlf true
git config --global core.safecrlf true

快照

文件的状态：raw -> changed -> staged -> commited -> pushed

git add => changed -> staged
git commit => staged -> commited
git push => commited -> pushed

git status => 显示 changed 和 staged 中的内容
git diff => 显示 change 和 raw 之间的对比
git log => 显示提交记录（commited 的版本历史）

  • git log --pretty=oneline => 每次提交只占一行
  • git log --pretty=oneline --max-count=2 => 最近几次提交
  • git log --pretty=oneline --since="5 minutes ago" => 最近几分钟的提交
  • git log --pretty=oneline --until="5 minutes ago" => 直到几分钟前的提交
  • git log --pretty=oneline --author=<username> => 某个人的提交
  • git log --pretty=oneline --all
  • git log --all --pretty=format:"%h %cd %s (%an)" => 格式化输出

git rm => 删除文件（raw/changed/staged -> staged）
git mv => 移动文件（raw/changed/staged -> staged）
git notes

状态回退：

  • changed 回退到 raw 状态 => git checkout . 或 git checkout <file-path>
  • staged 回退到 changed 状态 => git restore --staged . 或 git restore --staged <file-path> 或 git reset HEAD . 或 git reset HEAD <file-path>
  • commited 回退到上一个版本 => git revert HEAD --no-edit
  • commited 回退到历史版本 => git revert <hash> --no-edit
  • commited 无痕回退到历史版本 => git reset <hash>（不安全）

.gitignore

Git - gitignore Documentation (git-scm.com)

指定本地仓库中的多个目录或文件排除在外的一组模式串，称为 gitignore

有四种定义 gitignore 的方式：（优先级由高到低）
=> 命令行
=> /.gitignore
=> /.git/info/exclude 文件
=> core.excludesFile 配置变量

任意一个目录或文件路径形如：
=> 目录：/path/to/dir/
=> 文件：/path/to/file.ext

基本原理 => 若一条 gitignore 模式串匹配某个目录或文件路径的后缀或整个路径本身，那么该路径下的目录或文件被排除

通配符或规则：
? => 匹配 1 个任意字符
* => 匹配多个 [^/] 字符
prefix/** => 匹配 prefix 目录下的所有文件
prefix/**/suffix => 匹配 prefix/suffix 或 prefix/.../suffix
**/suffix => 匹配
** => 会视为 * ？
something => 匹配目录或文件路径的后缀
/something 或 path/to/something 或 /path/to/something => 匹配目录或文件的完整路径

子仓库

Git嵌套Git仓库|极客教程 (geek-docs.com)
git中submodule子模块的添加、使用和删除_git 设置submodule-CSDN博客

创建子模块

  • 创建 github 仓库
  • git submodule add <url> <path> => 添加子仓库（子模块）

克隆多模块仓库 => git clone <url> --recursive
子模块批量操作 => git submodule foreach [git pull/add/commit/push]

更新远程子模块的例子：（朴素的操作）

# 进入子模块进行 push 操作
cd <sub-module>
git add .
git commit -m "backup %date% %time%"
git push
# 主模块进行 push 操作
cd ..
git add .
git commit -m "backup %date% %time%"
git push

.gitmodule 文件的例子：

[submodule "obsidian-assets"]
path = assets
url = https://github.com/liangxiongsl/obsidian-assets

[submodule "docs-me"]
path = docs-me
url = https://github.com/liangxiongsl/docs-me

.git/config 文件的例子：

[core]
	repositoryformatversion = 0
	filemode = false
	bare = false
	logallrefupdates = true
	symlinks = false
	ignorecase = true
[remote "origin"]
	url = https://github.com/liangxiongsl/obsidian-vault
	fetch = +refs/heads/*:refs/remotes/origin/*
[branch "main"]
	remote = origin
	merge = refs/heads/main
[submodule "obsidian-assets"]
	active = true
	url = https://github.com/liangxiongsl/obsidian-assets
[submodule "docs-me"]
	active = true
	url = https://github.com/liangxiongsl/docs-me
