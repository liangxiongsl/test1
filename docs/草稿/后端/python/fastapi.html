from typing import Union
from fastapi import FastAPI

app = FastAPI()


@app.get("/")
def read_root():
    return {"Hello": "World"}


@app.get("/items/{item_id}")
def read_item(item_id: int, q: Union[str, None] = None):
    return {"item_id": item_id, "q": q}

装饰器

相关：python > 装饰器, github > REST, http > 状态码

fastapi 的装饰器 @app 用于给路由函数指定 path 和 request method
=> 其中 path 中的 {path-param} 指定了路径参数 path-param
=> 该路径参数可作为路由函数的形参

@app.<action>(path: string, response_model?: string, ...)

@app操作 <action> 用途
get(path: string) 指定某个 path 下的 get 请求接口方法
post(path: string) 指定某个 path 下的 post 请求接口方法
put(path: string) 指定某个 path 下的 put 请求接口方法
patch(path: string) 指定某个 path 下的 patch 请求接口方法
delete(path: string) 指定某个 path 下的 delete 请求接口方法
options()
head()
trace()

注：path 中可以用 {path-param} 指定路径参数 path-param

response_model: any => 指定响应的模型类
response_model_exclude_unset: bool => 响应中不包含未设置的默认键值对
response_model_exclude_defaults: bool
response_model_exclude_none: bool
response_model_include: set | list | tuple => 响应模型中包含某些键值对
response_model_exclude: set | list | tuple => 响应模型中不包含某些键值对

路径参数(path param)

相关：装饰器

路径参数 解释
{path-param} 匹配 [^/]+，并传递给路由函数的形参 path-param
{path-param:path} 匹配 .+，并传递给路由函数的形参 path-param
声明类型

为路径参数声明类型的例子：

@app.get("/items/{item_id}")
async def read_item(item_id: int):	# => 路由函数
    return {"item_id": item_id}

注：其中 item_id 会转换为 int 类型
注2：item_id 会被进行数据校验（由 Pydantic 完成）
注3：/docs 或 /redoc 路径下的接口文档会自动更新
注4：@app.get("/items/{item_id}") 会匹配 /items/[^/]*，若数据类型校验失败那么报错 => 不会继续匹配下一条路由

自定义的 Enum 枚举类也适合用作路径参数的类型

查询参数(query param)

路由函数声明了不属于路径参数的参数时，则 fastapi 将其视为路径参数

@app.post("/items/{item_id}")
def create_item(item_id: int, item: int):
    return {'item id': item_id, 'item': item}

=> 其中 item 会作为路径参数

注：请求 /example/path?q=123&q=456 的查询参数 q 被视为数组 [123, 456]

请求体(body param)

注：GET 方法不能发送请求体

路由函数中继承自 Pydantic.BaseModel 的形参的值，会由请求体 body param 的部分内容提供

例子：

class User(BaseModel):
    username: str
    password: str

@app.post('/test')
def example_func(user: User):
    return {'user is': user}

=> 请求体 {"username":"liangxiongsl","password":"1506218507"} 会填充到形参 user 上

例子2：

class User(BaseModel):
    username: str
    password: str

@app.post('/test/{param}')
def example_func(user1: User, user2: User):
    return {'user1 is': user1, 'user2 is': user2}

若请求体为 =>

{
 	"user1":{"username":"liangxiongsl","password":"1506218507"},
 	"user2":{"username":"travis","password":"2147483647"}
}

那么 key user1 会填充到同名的形参 user1（user2 同理）

更多：请求体 - 多个参数 - FastAPI (tiangolo.com)

数据校验，元数据

基本用法

fastapi.Query/Path/Body 用于给 query/path/body param 提供 数据校验 和 元数据
（或者显式定义路由函数的 query/path/body param）

数据校验选项 用途 补充
default: any 指定当前参数是否必选 ... / None / pydantic.Required / 不声明 => 必选
其他值 => 可选
min_length: int 指定字符串参数长度的下限 特定于字符串
max_length: int 指定字符串参数长度的上限 特定于字符串
regex: regex 指定字符串参数的匹配模式 特定于字符串
lt: int 当前参数小于某个值
gt: int 当前参数大于某个值
le: int 当前参数不大于某个值
ge: int 当前参数不小于某个值
元数据选项 用途 补充
alias: string 给请求参数指定别名 该别名用于发送客户端请求时
title: string 指定当前路由函数在文档中的 title
description: string 指定当前路由函数在文档中的 description
deprecated: bool 指定当前参数是否已弃用
embed: bool 是否期望单个请求体参数以键值对形式出现在请求体中 仅限 body param
examples: any[] 为 docs 或 redoc 文档添加参数的样例

例子：

@app.get('/test')
def example_func(q_url: str = Query(
    ...,
    alias='query-url',
    title='example title',
    description='example description',
    deprecated=False,
    min_length=3,
    max_length=100,
    regex=r'^(http|ftp|https):\/\/[\w\-_]+(\.[\w\-_]+)+([\w\-\.,@?^=%&:/~\+#]*[\w\-\@?^=%&/~\+#])?$'
)):
    return {'query url': q_url}

=> 示例请求：

http://localhost:8000/test?query-url=https://blog.csdn.net/qq_38819293/article/details/81570751

补充：形参 param: any = Path(...) 类似于 param: Annotated[any, Path(...)]

参数排序

将 * 作为路由函数的首个参数，那么后续参数可以自由排序
=> 参数的顺序对 fastapi 调用路由参数无关紧要

例子：

@app.get('/user/{uid}')
def example_func(*, uid: str = 'liangxiongsl', q: str):
    return {'user id': uid}
请求体-参数验证

pydantic 包下的 BaseModel 和 Field 用于定义请求体所需对象及其字段的数据校验

from pydantic import BaseModel, Field
from fastapi import Body
class User(BaseModel):
    name: str = Field('default username',title='uname title', description='username description',
                      min_length=5,max_length=15)
    password: str = Field('default password',title='upass title', description='password description',
                      min_length=5,max_length=15)

@app.post('/user')
def create_user(user: User = Body(embed=True)):
    return {"user": user}

多模型功能

用户模型的例子：

  • 输入模型 => 需要密码属性
  • 输出模型 => 不包含密码
  • 数据库模型 => 保存密码的哈希值，参考 OAuth2 实现简单的 Password 和 Bearer 验证

例子：user_model.py => 

from pydantic import BaseModel, EmailStr

class User_base(BaseModel):
    username: str
    email: EmailStr

class User_in(User_base):
    password: str
class User_out(User_base):
    pass
class User_in_db(User_base):
    hashed_password: str

def hasher(raw: str):
    return '<PASSWORD>' + raw

def save_user(user: User_in) -> User_in_db:
    ret = User_in_db(**user.dict(), hashed_password=hasher(user.password))
    return ret

index.py => 

@app.post('/user', response_model=User_out)
def create_user(user: User_in = Body(embed=True)):
    return save_user(user)
