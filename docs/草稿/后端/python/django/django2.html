基本配置

环境配置

  • pip install django==5.0.1 => 安装 5.0.1 版本的Django（该版本对应 3.10 以上的 python 版本）
  • django-admin => Django 脚手架（安装 Django 后会出现在 python/lib 目录下）
  • django-admin startproject <项目名> => 初始化项目

项目目录

理论

参考：一杯茶的时间，上手 Django 框架开发 - 知乎 (zhihu.com)

MTV

Django MTV

  • M（Model）：创建数据模型，并执行数据库迁移
  • T（Template）：写出基本的 Django 模板，并向视图中传入数据
  • V（View）：在视图中访问数据库，实现业务逻辑，渲染模板，并接入路由表
  • U（Url）：路由表

App

app

  • 内置app——框架自带的应用，如：admin（后台管理）、auth（身份鉴权）、sessions（会话管理）
  • 第三方app
  • 自定义app

  • python manage.py startapp <app名> => 创建自定义 app => <项目名> 目录下将创建一个app，结构如下：
exampleApp               // 自定义应用目录
├── __init__.py          // 初始化模块
├── admin.py             // 后台管理配置
├── apps.py              // 应用配置
├── migrations           // 数据库迁移文件目录
│   └── __init__.py      // 数据库迁移初始化模块
├── models.py            // 数据模型
├── tests.py             // 单元测试
└── views.py             // 视图
  • <项目名>/settings.py 文件中的 INSTALLED_APPS 列表 => 配置 app，如：
INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'exampleApp',
]

View

（涉及文件——urls.py, views.py）

视图

  • FBV（Function-based View，基于函数的视图）
  • CBV（Class-based View，基于类的视图）
    注：<app名>/views.py 文件下的一个函数就是一个 FBV

路由系统——将请求的 path 转发到视图

  • 全局路由
  • 子应用路由

路由配置：

  1. 在 <项目名>/urls.py 中的 urlpatterns 进行配置——path(<route>, <view>, <name>)
    注：
  • route——访问的路由，如 / 或 admin/
  • view——转发的路由或视图，如 include('exampleApp.urls')、views.exampleFBV
  • name——指定路由的名称，在模版中会引用（可选）

Template

（涉及目录——templates/）

模板类似于 jsp，用于 html 嵌入代码

(1) 表达式插值

  • {{ value }}——单个变量
  • {{ dict.key }} / {{ obj.attr }}——取字典的键值 或 对象的属性
  • {{ list.4 }}——取列表的某个元素
    注：其中的变量来自于上下文字典——Context dictionary（一个 json 对象的字符串）

(2) 条件语句：例如

{% if is_true %}
  <h1>It is true!</h1>
{% else %}
  <h1>It is false!</h1>
{% endif %}

(3) 循环语句，例如：

{% for elem in list %}
  <p>{{ elem }}</p>
{% endfor %}

视图中可以使用 render(request, <template_path>, <context>) 来使用上下文 context 对 template 进行渲染，得到渲染后的内容
注：Django 的模板搜索机制是将项目中的所有 templates 目录整合在一起？
注2：<template_path>——例如，path/to/template

Model

（涉及文件或目录——models.py, migrations/）

优势

  • 由于高度解耦的设计，可轻松切换各种关系型数据库（默认的 SQLite，可选 MySQL、PostgreSQL、Oracle 等等）
  • 强大的 ORM（Object Relation Mapping，对象关系映射）模块，使得用 Python 操作数据库非常轻松，免去了使用 SQL 的麻烦
  • 优秀的数据库迁移机制（Migration），修改数据模式（Schema）比较方便，能够适应不断变化的功能需求

ORM——将面向对象代码映射到 sql 语句（操作数据表）

  • <Model>(field1=<field1>, field2=<field2>, ...) => 创建 model 对象
  • <model>.save() => 插入 model 对象
  • <model>.objects.get(id=1) => 条件查询 model 对象
  • <model>.objects.all() => 获取某个 model 的所有对象

数据库迁移（migration）——将模型映射为 sql 语句（建表/修改表结构）=>执行迁移

  • 在 <app>/models.py 下定义 model，例如：
class Post(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()

    def __str__(self):
        return self.title
  • python manage.py makemigrations => 生成迁移文件于 <app>/migrations/ 目录
  • python manage.py migrate => 执行迁移 => 数据库表更新

python manage.py createsuperuser => 创建超级用户

RESTful

在前后端分离的应用模式中，我们通常将后端开发的每个视图都称为一个接口，或者API，前端通过访问接口来对数据进行增删改查。

restful是一种后端API接口规范

(1) http method

http method 含义 对应的 sql 命令
GET 从服务器取出资源 select
POST 在服务器中新建资源 create/insert
PUT 在服务器中更新资源(客户端提供改变后的完整资源) update
DELETE 从服务器中删除资源 delete
PATCH 在服务器中更新资源(客户端提供改变的属性) update
HEAD 获取资源的元数据
OPTIONS 获取信息，关于资源的哪些属性是客户端可以改变的

(2) 过滤信息，如：

?limit=10：指定返回记录的数量
?offset=10：指定返回记录的开始位置。
?page=2&per_page=100：指定第几页，以及每页的记录数。
?sortby=name&order=asc：指定返回结果按照哪个属性排序，以及排序顺序。
?animal_type_id=1：指定筛选条件

(3) 状态码：

状态码 标识 对应method 含义
200 OK GET 服务器成功返回用户的请求数据
201 CREATED POST/PUT/PATCH 新建或修改数据成功
202 Accepted 请求已经进入后台排队（异步任务）
204 NO CONTENT DELETE 删除数据成功
400 INVALID REQUEST POST/PUT/PATCH 非法请求
401 Unauthorized 用户没用权限
404 Not FOUND GET 用户请求的是不存在的记录，服务器没有进行操作
406 Not Acceptable GET 用户请求格式错误
410 Gone GET 用户请求的资源已被删除
422 Unprocesable entity POST/PUT/PATCH 创建对象时，发送验证错误
500 INTERNAL SERVER ERROR 服务器发生错误

(4) 错误处理

如果状态码是4xx，服务器就应该向用户返回出错信息。一般来说，返回的信息中将error作为键名，出错信息作为键值即可，如：

{
    error: "Invalid API key"
}

(5) 返回结果——针对不同操作，服务器向用户返回的结果应该符合以下规范。服务器返回的数据格式，应该尽量使用JSON

GET /collection：返回资源对象的列表（数组）
GET /collection/resource：返回单个资源对象
POST /collection：返回新生成的资源对象
PUT /collection/resource：返回完整的资源对象
PATCH /collection/resource：返回完整的资源对象
DELETE /collection/resource：返回一个空文档

DRF

Django REST framework 框架是一个用于构建Web API 的强大而又灵活的工具

特点：

  • 提供了定义序列化器Serializer的方法，可以快速根据 Django ORM 或者其它库自动序列化/反序列化
  • 提供了丰富的类视图、Mixin扩展类，简化视图的编写
  • 多种身份认证和权限认证方式的支持
  • 直观的 API web 界面

pip install djangorestframework => 安装 DRF

序列化/反序列化

  • 序列化——响应的python对象转化为响应报文的json字节流
# 1、拿到对象
student = Student.oject.all()
# 2、将对象交给序列化类
serializer = StudentSerializer（student）
# 3、字典转json字节流
JSONRenderer().render(serializer.data())
  • 反序列化——请求body的json字节流转化为python对象
# 1、json字节流转为字典
data = JSONParser().parse(BytesIO(content))
# 2、通过serializer将字典转对象
serializer = StudentSerializer（data=data）
# 3、校验对象数据
serializer.is_valid()   #校验通过返回True,失败返回False
# 4、保存对象
serializer.save()
