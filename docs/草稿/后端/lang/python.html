模块

__name__ => 获取模块名，正在运行的模块为 __main__？
import <module> => 导入模块 => 执行模块的内容，将模块中的内容放到命名空间 <module> 中
from <module> import <content> => 导入模块的内容到当前模块的命名空间中
from <module> import * => 导入模块的所有内容到当前模块的命名空间中（这些内容不以下划线 _ 开头）

importlib.reload(<module>) => 重新导入模块

模块下的 __pycache__/<module>.<py-version>.pyc 文件是模块的编译文件

dir(<module>): str[] => 返回已定义的标识符（如：函数名，模块名，类名，变量名）

模块搜索路径
  1. python 解释器会在 sys.builtin_module_names 中寻找要导入的模块 <module>
  2. 在 sys.path 中的每个目录下寻找 <module>.py 文件

注：sys.path 中的目录由 当前脚本目录、PYTHONPATH、site-packages 等提供

包

包由三部分组成：__init__.py（必选），模块，子包
=> 包也是一个名为 __init__.py 的模块

基本import语法

import path.to.package => 导入模块 path.to.package.__init__
import path.to.module => 必须用 path.to.module.<something> 引用模块的内容
from path.to.package import <content | module | package> => 导入的可能是 3 种东西 => 包中 __init__.py 的内容、包中的模块、子包
from path.to.module import <content> => 导入模块的内容
from path.to.package import * => 默认导入包下的所有模块 => 若包下的 __init__.py 定义了 __all__: str[]，那么会导入模块数组 __all__ 中指定的所有模块
from path.to.module import * => 导入模块的所有内容

绝对导入

from path.to.something import ... => 相对 __main__ 模块所在包的路径中导入

相对导入

from . import <something>
from .<something> import <something>
from .. import <something>
from ..<something> import <something>

装饰器

@decorator 类似于 java 中的注释

像一顶漂亮的装饰帽一样，将它放在一个函数的上方（我猜测这个术语的命名就是这么来的）

语法糖将 @decorator: (func: function) => function 放到函数 example_func 前面
=> 等价于 在 example_func 定义后，调用 example_func = decorator(example_func)
注：decorator 也可以是一个表达式（如：函数的返回值，该值也是一个装饰器函数）

装饰器语法有几个用处：

  • 声明函数在定义前的行为
  • 声明函数在调用前的行为
  • 有条件的调用函数（比如：在调用前检查函数是否“合法”或"合理"）

例子1：=> "不带参数"的装饰器

def my_decorator(func):
	# do something before some functions defined (1)
    def wrapper(*args, **kwargs):
		# do something before some functions called
        return func(*args, **kwargs)
	# do something before some functions defined (2)
    return wrapper

# 使用装饰器
@my_decorator
# => 在 my_func 定义完成后，调用 my_func = my_decorator(my_func)
def my_func(*args, **kwargs):
	...

例子2： => "带参数"的装饰器

def my_decorator(arg1, arg2, ...):
	# do something before some functions defined (1)
    def decorator(func):
		# do something before some functions defined (3)
        def wrapper(*args, **kwargs):
			# do something before some functions called
            return func(*args, **kwargs)
		# do something before some functions defined (4)
        return wrapper
	# do something before some functions defined (2)
    return decorator

# 使用装饰器
@my_decorator(<arg1>, <arg2>, ...)
# => 在 my_func 定义完成后，调用 my_func = my_decorator(<arg1>, <arg2>, ...)(my_func)
def my_func(*args, **kwargs):
	...

解包

函数的参数，或 类的构造函数的参数 都能看作字典 => func_dict, arg_dict
=> 那么任意一个上述字典的子集字典 sub_dict 可以简单地拷贝给 func_dict 或 arg_dict =>

  • 例子1：
def func(a:str, b:str, d:str):
    print(a, b, d)

sub_dict = { 'a':'foo', 'b':'bar' }
func(**sub_dict, d='giegie')
# => 输出 foo bar giegie
  • 例子2：
class Nodes:
    a: str
    b: str
    d: str
    def __init__(self, a:str, b:str, d:str):
        self.a = a
        self.b = b
        self.d = d
    def __str__(self):
        return f'{self.a} {self.b} {self.d}'

sub_dict = { 'a':'foo','b':'bar', 'c': 'orther arg' }
print(Nodes(**sub_dict, d='giegie'))
# => 输出 foo bar giegie

切片

语法 => obj[leftC : rightO] 或 obj[leftC : rightO : step]

  • leftC => 一个整数，指定左闭区间；为空时视为 0
  • rightO => 一个整数，指定右开区间；为空时视为 len
  • : step => step 为一个正整数，指定切片步长，默认为 1
