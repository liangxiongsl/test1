原理

SSM => Spring + Spring MVC + MyBatis

  • Spring MVC DispatcherServlet：拦截请求，根据其 url 选择对应的 Controller 进行处理
  • Spring IoC 和 依赖注入：获取 Service 层的实例，并调用其相应方法实行业务逻辑处理
  • Service 层：调用 MyBatis Mapper 接口进行数据库操作
  • MyBatis：获取SQL语句的返回
  • Service 层：将 MyBatis 返回的结果封装成业务逻辑需要的数据类型，返回给 Controller
  • DispatcherServlet：获取 Controller 的返回，渲染视图并返回给用户

项目结构

(1) 源码结构：（src/main/java 目录下，分为四种包）

  • dao(Data Access Object)：支持访问数据库的设计模式，将数据存储逻辑和业务逻辑解耦
    • 数据源(Data Source)：
    • 实体类(Entity Class)：
    • DAO接口：Mapper 接口
    • DAO实现类：Mapper xml
  • pojo(Plain Old Java Object)：框架无关的类，一般只含有私有字段和getter/setter；不实现接口、继承、依赖特定框架
    • 也可以认为是 dao 的实体类
  • bean：只有私有字段、getter/setter、无参构造等；用于持久化
  • service：
    • service接口：与 Mapper 接口相同（除了接口名）
    • service实现类：包含 Mapper 接口对象，实现 service 接口
  • Controller：

Controller <=> Service <=> dao/pojo

注：dao/pojo/bean 三者类似；pojo/bean 更加类似

Spring MVC

SpringMVC中常用注解与使用方法_mvc中的模型注解怎么用-CSDN博客

注解 注释的对象 作用 注释参数 补充
@RequestMapping 控制器类/方法 建立请求url与控制器方法的对应关系 value,mothed,params
@GetMapping 控制器类/方法 等于 @RequestMapping(method = RequestMethod.GET)
@PostMapping 控制器类/方法 等于 @RequestMapping(method = RequestMethod.POST)
@RequestParam 形参 用某个请求参数初始化该形参 value
@RequestHeader 形参 用某个请求消息头初始化该形参 value
@RequestBody 形参 用请求体初始化该形参
@PathVariable 形参 用某个以{value}指定的请求path初始化该形参 value
@ModelAttribute 控制器方法 当前控制器方法会在其他方法执行前执行
@ModelAttribute 形参 用指定数据舒初始化该形参 value
@CookieValue 形参 用指定cookie初始化该形参 value
@SessionAttributes 用于多次执行控制器方法间的参数共享？ value

注：

@RequestMapping

@RequestMapping(value = "/hello",method = RequestMethod.GET,params = {"username"})：用于建立请求URL和处理请求方法之间的对应关系

  • value：指定url请求的 path
  • method：指定请求的方式
  • params：限定请求参数
    注：
  • 作用在类上 => 对应第一级的path
  • 作用在方法上 => 对应第二级的path

@GetMapping

等于 @RequestMapping(method = RequestMethod.GET)

@PostMapping

等于 @RequestMapping(method = RequestMethod.POST)

@RequestParam

@RequestParam(value = "username", required = true)：作用在控制器中某个方法的某个字符串形参，用于调用该方法时初始化该形参

  • value：请求参数名
  • require?：指定该请求参数是否必须存在，否则报错；默认为 true

@RequestHeader

@RequestHeader(value = "Accept", required = true)：作用在控制器中某个方法的某个字符串形参，用于获取某个请求消息头

  • value：消息头名称
  • required?

@RequestBody

@RequestBody(required = true)：作用在控制器中某个方法的某个字符串形参，用于获取请求体（形如 key1=value1&key2=vaule2）

  • require?：指定请求体是否必须存在（get 请求不适用：require=true 时报错；require=false时得到 null）

@PathVariable

@PathVariable(value = "id")：作用在控制器中某个方法的某个字符串形参，用于绑定请求 path 中的占位符

  • value：占位符名
    如：
@RequestMapping("/book/{id}")
public String test(@PathVariable(value = "id") String id){
    System.out.println(id);
    return "success";
}

@CookieValue

@CookieValue(value = "JSESSIONID")：作用在控制器中某个方法的某个字符串形参，用于获取某个cookie

  • value：cookie名
  • require?

@ModelAttribute

@ModelAttribute => 作用于控制器的某个方法上 => 当前方法会在其他方法执行前执行
@ModelAttribute(value = "") => 作用于控制器的某个方法的某个参数上 => 获取指定的数据给参数赋值

@SessionAttributes

@SessionAttributes(value = "", type = "")：用于多次执行控制器方法间的参数共享？

  • value：指定存入的属性名称
  • type：指定存入的数据类型

注解

用于创建对象

把资源让Spring管理，相当于在 xml 中配置一个 bean

注解 特点
@Component
@Controller 在表现层
@Service 在业务层
@Repository 在持久层

公共参数——value：指定 bean 的id，默认为当前的类名（首字母小写）

用于注入数据

@Autowired：自动按照类型注入。当使用注解注入属性时，set方法可以省略。它只能注入其他bean类型；当有多个类型匹配时。使用要注入的对象变量名称作为bean的id，在spring容器中查找，找到了注入成功，找不到就报错

@Qualifier：在自动按照类型注入的基础上，再按照 Bean 的 id 注入；
给字段注入 => 必须与 @Autowired 一起使用
给方法参数注入 => 可单独使用
属性 value：指定 Bean 的 id

@Resource：直接按照 Bean 的 Id 注入。只能注入其他器 Bean 类型
属性 name：指定 Bean 的 id

@Value：注入基本数据类型和String数据类型

用于改变作用范围

@Scope：指定 Bean 的作用范围（可作用于业务层的类）
属性 value

  • singleton
  • prototype
  • request
  • session
  • global-session
与生命周期相关
  • @PostConstruct
  • @PreDestroy
Spring5

@Configuration：指定当前类是一个 Spring 配置类，当有容器时会从该类上加载注解。获取容器是使用AnnotationApplicationContext（有@Configuration注解的类.class）

@ComponentScan：用于指定spring在初始化容器时要扫描的包。作用和在spring的xml配置文件找那个的<context : component-sacn base-package="com.dynamic"/>

Bean：作用于方法上，表明使用此方法创建一个对象，并放入 Spring 容器中

Import：用于导入其他配置类

<mapper namespace="com.dynamic_basics.dao.IUserDao">
 
    <!--使用配置的方式解决Java实体类与数据库列名不一致的问题-->
    <resultMap id="userMap" type="com.dynamic_basics.domain.User">
        <!--主键字段-->
        <id property="userId" column="id"></id>
        <!--非主键字段-->
        <result property="userName" column="username"></result>
        <result property="userAddress" column="address"></result>
        <result property="userSex" column="sex"></result>
        <result property="userBirthday" column="birthday"></result>
    </resultMap>
 
    <!--查询所有 id使用方法名-->
    <select id="findAl l" resultType="com.dynamic_basics.domain.User" resultMap="userMap">
        select * from user
    </select>
 
    <!--保存用户-->
    <insert id="saveUser" parameterType="com.dynamic_basics.domain.User">
        <!--参数使用了ognl表达式-->
        insert into user(username,address,sex,birthday)values(#{username},#{address},#{sex},#{birthday});
    </insert>
 
    <!--更新用户-->
    <insert id="updateUser" parameterType="com.dynamic_basics.domain.User">
       update user set username=#{username},address=#{address},sex=#{sex},birthday=#{birthday} where id=#{id}
    </insert>
 
    <!--删除用户-->
    <delete id="deleteUser" parameterType="int">
        delete from User where id=#{id}
    </delete>
 
    <!--根据id查询用户-->
    <select id="findById" parameterType="int" resultType="com.dynamic_basics.domain.User">
        select * from user where id=#{id}
    </select>
 
    <!--根据名称模糊查询-->
    <select id="findByName" resultType="com.dynamic_basics.domain.User" parameterType="String">
        select * from user where username like #{name}
    </select>
 
    <!--获取用户的总记录数-->
    <select id="findTotal" resultType="int">
        SELECT COUNT(id) FROM `user`;
    </select>
