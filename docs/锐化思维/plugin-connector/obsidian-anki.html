obsidian--anki

概述

obsidian 与 anki 的 B/S 联动：

  • ObsidianToAnki/Obsidian_to_Anki：将文本/Markdown 文件中的抽认卡添加到 Anki 的脚本 (github.com)
  • alex/anki-connect - anki-connect - FooSoft Productions
    注：anki-connect 通过开启服务器来开放 http api；obsidian to anki 通过 http 来与 anki-connect 进行通信

安装 anki-connect：

  • 工具->插件->获取插件，使用 id 2055492159 安装插件
  • 设置插件 => webCorsOriginList key 中添加 app://obsidian.md（允许 obsidian 跨域）

双向通信/交流：

  • anki->obsidian => 使用 obsidian2 的 uri
  • obsidian->anki => 使用 anki-connect 的 http api

不足：obsidian 似乎不能动态侦听 anki 数据库的变化

核心

笔记：

  • 块笔记，例子：
# 方式1
BEGIN
<note type>
Front: <front>
Back: <back>
Tags: <tag1> <tag2> ...
END
# 方式2（第一个字段可以省略 key）
BEGIN
<note type>
<front>
Back: <back>
Tags: <tag1> <tag2> ...
END
  • 行内笔记
STARTI [<note type>] <front> Back: <back> ENDI

注：更新任何 note type 时，应在 obsidian to anki 的设置中执行 Regenerate Note Type Table 命令

标签：

  • 基于文件：
// (1)
FILE TAGS: <tag1> <tag2> ...
// (2)
FILE TAGS
<tag1> <tag2> ...
  • 基于笔记：
BEGIN
<note type>
<note fields>
Tags: <tag1> <tag2> ...
END
  • 基于目录 => 在 设置->NoteTypeSettings.FolderSettings 中设置某个目录的标签

牌组：

  • 基于文件：
# 方式1
TARGET DECK
<deck>
# 方式2
TARGET DECK: <deck>
  • 基于目录 => 在 设置->NoteTypeSettings.FolderSettings 中设置某个目录所属的牌组

删除笔记：

DELETE
ID: <id>

cloze 语法：

  • 支持 anki 原生的 {{c<num>::<text>}}
  • 启用 Defaults.CurlyCloze => 支持 {<text>}, {<num>:<text>}, {<num>|<text>}, {c<num>:<text>}, {c<num>|<text>} 5 种简化的插空格式
    注：更新完形填空后可能出现空卡片，需要在 anki 中执行 工具->清空卡片... 命令
    注2：第 2 种语法不会翻译数学代码中的花括号 {}

Frozen Fields => 给某种笔记的一些字段追加内容：

FROZEN - <note type>:
<filed1-key>: <filed1-content>
<filed2-key>: <filed2-content>
...

=> 例子：

FROZEN - Cloze:
<br><a href='https://github.com/liangxiongsl'><img src="https://avatars.githubusercontent.com/u/62007734?v=4" style="width: 30px"></a>

注：第一个字段可以省略 <filed1-key>

特性

  • markdown，latex
  • 链接到 obsidian，如：[[wiki-link]]
  • obsidian集成
    - 链接支持 => wiki url/文件链接
    - 嵌入支持 => obsidian风格嵌入 音频/图片，
    如：![[<path/to/audio>]]
    - 标签支持
    注：通过 设置->Defaults 进行相关配置
    注：应及时删除未使用的媒体

笔记定义风格

官方推荐的卡片定义风格：

  • RemNote 风格
  • heading-paragraph 风格
  • 问答风格
  • neuracache 风格
  • ruled 风格
  • markdown table 风格
  • cloze 段落风格

RemNote 风格

  • regex => ^(.*[^\n:]{1}):{2}([^\n:]{1}.*)
  • 定义方式：
<question>::<anser>
<!--ID: 1716449891200-->

heading-paragraph 风格

  • regex => ^(.*[^\n:]{1}):{2}([^\n:]{1}.*)
  • 定义方式：
# <question 1>
<answer 1>
<!--ID: 1716449891200-->

# not question title
## <question 2.1>
<answer 2.1>
<!--ID: 1716449891215-->

## <question 2.2>
<answer 2.2>
<!--ID: 1716449891221-->

问答风格：

  • regex => ^Q: ((?:.+\n)*)\n*A: (.+(?:\n(?:^.{1,3}$|^.{4}(?<!<!--).*))*)
  • 定义方式：
Q: <question>
A: <answer>
<!--ID: 1716450805491-->

neuracache 风格：

  • regex => ((?:[^\n][\n]?)+) #flashcard ?\n*((?:\n(?:^.{1,3}$|^.{4}(?<!<!--).*))+)
  • 定义方式：
<question> #flashcard
<answer>
<!--ID: 1716451118005-->

ruled 风格

  • regex => ((?:[^\n][\n]?)+\n)-{3,}((?:\n(?:^.{1,3}$|^.{4}(?<!<!--).*))*)
  • 定义方式：
<question>
---
<answer>
<!--ID: 1716451275565-->

markdown table 风格

  • regex => \|([^\n|]+)\|\n\|(?:[^\n|]+)\|\n\|([^\n|]+)\|\n?
  • 定义方式：
| <question> |
| ---------- |
| <answer>   |
<!--ID: 1716451451558-->

cloze 段落风格

  • regex => ((?:.+\n)*(?:.*{.*)(?:\n(?:^.{1,3}$|^.{4}(?<!<!--).*))*) 或 ((?:.+\n)*(?:.*==.*)(?:\n(?:^.{1,3}$|^.{4}(?<!<!--).*))*)
  • 定义方式，例子：
With {2:CurlyCloze} enabled, you can also use the {c1|easier cloze formatting},
but of course {{c3::Anki}}'s formatting is always an option.
<!--ID: 1716451639477-->

注：笔记模板必须继承 Cloze
注2：推荐打开 CurlyCloze 设置

anki connect

alex/anki-connect - anki-connect - FooSoft Productions

默认开放端口：8765

http api 使用 json 通信：（post）

  • 请求接口：
{
	action: str,
	params:? dict,
	version?: int,
	key?: str
}
  • 响应接口：
{
	result: any,
	error: str
} | any

		params: {
			card: int,

			newValues: str[]
		},
		result: bool[]
	},
	// (4) 暂停多个卡片

		params: {
			cards: int[],
		},
		result: bool
	},

		params: {
			cards: int[],
		},
		result: bool
	},
	// (7) 检验卡片是否暂停
	suspended: {
		params: {
			card: int,
		},
		result: bool
	},
	// (8) 检验多个卡片是否暂停
	areSuspended: {
		params: {
			cards: int[],
		},
		result: bool[]
	},
	// (9) 检验多个卡片是否到期（需要复习）
	areDue: {
		params: {
			cards: int[],
		},
		result: bool[]
	},
	// (10)
