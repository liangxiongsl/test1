// 定义
class MyPluginSettingTab extends PluginSettingTab{

	constructor(app: App, plugin: MyPlugin_lx) {
		super(app,plugin);
		this.plugin = plugin
	}

	display(): void {
		const { containerEl } = this

	}
	
	hide(): any {
		const { containerEl } = this
		containerEl.empty()
	}
}
// 注册
this.addSettingTab(new MyPluginSettingTab(this.app, this))

注：一个插件只提供一个设置 tab ？

将 PluginSettingTab 类对象的 containerEl 暴露给 Setting 类对象，就能为前者添加配置项

例子2：Setting 类的使用

class MyPluginSettingTab extends PluginSettingTab{
	setting: Record<string, Setting>
	githubUrl: string
	description: string
	constructor(app: App, plugin: Plugin) {
		super(app, plugin);
		const { containerEl } = this
		this.setting["github url"] = new Setting(containerEl)
			.setName("github url")
			.addText((text) => {
				text.onChange((value) => { this.githubUrl = value })
			})
		this.setting["description"] = new Setting(containerEl)
			.setName("描述")
			.addTextArea((text) => {
				text.onChange((value) => { this.description = value })
			})
	}

	display(): void {
	}
}

通常会将配置数据存储在插件目录下的 data.json，并使用 Plugin 类的 loadData() 和 saveData(data) 方法进行配置的读取和存储

例子3：持久化配置数据

// 配置接口
interface MyPluginSettings{
	githubUrl: string
	description: string
}
// 默认配置
const DEFAULT_SETTINGS: Partial<MyPluginSettings> = {
	githubUrl: "liangxiongsl",
	description: "niu bi"
}
export default class MyPlugin extends Plugin {
	// 配置对象
	settings: MyPluginSettings
	async loadSettings(){
		this.settings = Object.assign({}, DEFAULT_SETTINGS, await this.loadData())
	}
	async saveSettings(){
		await this.saveData(this.settings)
	}

	async onload() {
		// 初次加载配置
		await this.loadSettings()
	}
}

// 配置的前端代码
class MyPluginSettingTab extends PluginSettingTab{
	plugin: MyPlugin

	constructor(app: App, plugin: MyPlugin) {
		super(app, plugin);
		this.plugin = plugin
	}

	display(): void {
		const { containerEl } = this
		new Setting(containerEl)
			.setName("github url")
			.addText((text) => {
				text
					.setValue(this.plugin.settings.githubUrl)
					.onChange((value) => this.plugin.settings.githubUrl = value )
			})
			.setDesc("您的url")
		new Setting(containerEl)
			.setName("描述")
			.setDesc("您的描述")
			.addTextArea((text) => { text
				.setValue(this.plugin.settings.description)
				.onChange((value) => this.plugin.settings.description = value )
			})
		new Setting(containerEl)
			.addButton((btn) => btn
				.setButtonText("重新加载")
				.onClick(async (eve) => { this.hide(); await this.plugin.loadSettings(); this.display() } )
			)
			.addButton((btn) => btn
				.setButtonText("保存")
				.onClick(async (eve) => await this.plugin.saveSettings() )
			)
	}
	hide(): void {
		const { containerEl } = this
		containerEl.empty()
	}
}

				text.onChange((value) => {
					this.result = value
				}))
		new Setting(contentEl)

					.setCta()
					.onClick(() => {

					})
			)

		let { contentEl } = this

	}
}

工作区(workspace)

参考：Workspace
相关：视图(views)

工作区 workspace 是代表主屏幕的树形结构，其中每个节点都是 WorkspaceItem

WorkspaceItem 分为 WorkspaceParent 和 WorkspaceLeaf 两种

  • WorkspaceParent 可以包含子 item；分为 split 和 tabs 两种
    • split => 子 item 以分页的形式作为其儿子（item 以某个方向依次布局）
    • tabs => 子 item 以百叶窗的形式作为其儿子（每次只显示一个 item）
  • WorkspaceLeaf 则不能包含子 item => 代表一个显示内容的窗口，而 leaf 的类型决定内容如何显示

workspace 拥有三个特殊的子 item

  • left split => tabs => 多个 leaf
  • root split => split, 其他 leaf
  • right split => tabs => 多个 leaf

注：有两种方式使 leaf 不显示：

  • <leaf>.detach()
  • workspace.detachLeavesOfType(<view_type>)

例子：

// (1) 注册自定义 view
this.registerView("my_view_id", (leaf) => new MyView(leaf) )  
const { workspace } = this.app  
const leaf = workspace.getLeaf(true)  
// (2) 将新的 leaf 与自定义 view 绑定
leaf.setViewState({type: "my_view_id", active: true})  
// (3) 显示 leaf
workspace.revealLeaf(leaf)

主题

主题开发流

  • <myVault>/themes => 插件位置
  • 插件文件 => *.css 或 包含 theme.css 和 manifest.json 的目录
  • 初始插件 => obsidianmd/obsidian-sample-theme: A template containing everything you need for creating your own Obsidian theme. (kkgithub.com)

.theme-dark / .theme-light => 两种主题样式
body => 两种主题的共同样式
:root => 用于定义变量

在使用中发现 css 样式 => 在控制台中的 sources 下找到 app.css，可以找到 ob 中的所有样式

重写内建的 css 变量 => 在 :root / body / .theme-dark / .theme-light 下重写 css 变量

避免 !important 声明

嵌入字体和图像

url("data:<MIME_TYPE>;base64,<BASE64_DATA>") => 返回加密的数据

  • MIME_TYPE => 参考 Common MIME types - HTTP | MDN (mozilla.org)
  • BASE64_DATA => base64 加密过的数据

用于数据加密的网站：

  • 字体加密
    => Woff2Base => 用于 WOFF2 字体文件
    => Aspose => 支持大部分字体格式
  • 图像加密
    => WebSemantics => 支持 JPEG, JPG, GIF, PNG, SVG 格式图像
    => Base64 Guru => 支持各种图像格式
    => Yoksel URL-encoder for SVG => 用于 svg 文件优化
