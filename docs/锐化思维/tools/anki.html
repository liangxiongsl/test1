核心

概念

涉及的内容：

  • anki => card, deck

内容介绍

  • note type/template => 定义了笔记字段的结构，通常有 front/back 字段 => 有 basic, Basic(and reversed card), Basic(optional reversed card), Basic(type in the answer), Cloze, Image Occlusion 6 种标准笔记模板

  • field => 字段

  • note => 笔记模板的实例
    注：通过继承标准笔记模板，才能自定义 note template

  • card type/template => 继承一个 note template 接口，定义了多个拥有 front/back/styling 字段的内部接口

  • card => 卡片模板的实例

  • deck => 牌组(树形结构)，支持挂载多个 card

  • collection => notes, cards, decks, tags, Review History 的集合

注：note template 定义后端，card template 定义前端，card 是要学习的对象，deck 是学习对象的集合
注2：一个 note template 和一个 card template 的实现，可以创造一个 note 和多个 card

卡片记忆程度的评价 => again, hard, good, easy

flag => 每个 card 都可以有一个 flag
tag => 每个 note 都可以由多个 tag
注：有 marked 标签的 note 在复习对应的 card 时会有星号（表示这个 note 很重要）

行为

笔记/卡片的搜索 => <key>:<content>

  • flag => 根据标记搜索
  • tag => 根据标签搜索
  • note => 根据笔记模板搜索
  • deck => 根据牌组搜索，如：deck:current, deck:test-deck
  • is => 根据卡片状态搜索，支持 new/learn/review/suspended/buried
  • 自定义筛选器

note template(笔记模板)

basic

分为四类：

  • basic => 根据题目进行回答，自行评价掌握的熟练度；生成 正面->反面 卡片
  • basic(and reversed card) => 根据题目进行回答，自行评价掌握的熟练度；正面->反面 和 反面->正面 两个卡片
  • basic(optional reversed card) =>
  • basic(type in the answer) => 根据 Front 中的题目，输入 Back 预期的答案

笔记模板支持的字段：Front, Back, Add Reverce（opt）

cloze

完形填空 => 对部分文本挖空，并回忆文本的内容，然后自行判断记忆程度

添加完形填空：

  • 卡片模板中 {{cloze:<note-field>}} 指定了需要编译笔记模板的 <note-field> 字段中的 {{c<num>::<content>}}
    注：根据 <num> 的不同数字的个数，卡片模板可以添加多个卡片
    注2：完形填空 cloze 只有一个卡片模板

笔记模板支持的字段：

  • 可编辑字段：Text, Back Extra

笔记模板的指令：

  • c<num>::<content> => 为指定的组别添加填空，如 {{c1::example}}；每个组别会有一个 c<num> 字段
  • c<num>::<content>::<tip> => 为指定的组别添加填空，并添加提示（否则提示内容为 ...）
    注：指令支持嵌套，如：{{c1::Canberra was {{c2::founded}}}} in 1913 => [...] in 1913 和 Canberra was [...] in 1913

卡片模板的指令：

  • cloze:<note-field> => 编译 <note-field> 笔记字段中的指令

相关快捷键：

  • ctrl+shift+alt+c => 在当前组别中添加新的填空
  • ctrl+shift+c => 新建组别，并添加填空
image occlusion

图像遮挡（可以使用 正方形/圆形/多边形 进行遮挡回忆） => 对部分图像进行遮挡，并遮挡部分的内容，然后自行判断记忆程度

注：note 对应的卡片的数量为遮挡图形的个数

笔记模板支持的字段：

  • 可编辑字段：Header, Back Extra, Comments
  • 其他字段：Occlusion, Image

card template(卡片模板)

卡片模板 => 具有 Front, Back, Styling 三个字段，分别表示 卡片正面、卡片反面、样式

注：Front 和 Back 字段定义的内容在同一个 html 中
注2：卡片模板与生成卡片的数量密切相关

指令

指令：

  • 字段替换：
    • {{<note-field>}} => 内容替换为 <note-field> 的值
    • {{hint:<note-field>}} => 首先显示 <note-field> 的标识符，点击后显示 <note-field> 的值
    • {{text:<note-field>}} => 不解释 <note-field> 的 html
    • {{furigana:<note-field>}} => 支持对 <note-field> 中的 <文本>[<上标>] 进行编译，效果如同：上标文本 或 梁
  • 特殊字段替换：
    • {{FrontSide}} => 正面的内容（仅限于背面）
    • {{Tags}} => note tags
    • {{Type}} => note template
    • {{Deck}} => 卡片所属的 deck
    • {{Subdeck}} => 卡片的 sub deck
    • {{CardFlag}} => 卡片的 flag
    • {{Card}} => card template
  • 文本转语言：
    • {{tts <lang>:<note-field>}} => 以某种语言朗读某个笔记字段，如：{{tts en_US:note_field}}，{{tts zh_CN:note_field}}
    • {{tts <lang> voices=<voices>:<note-field>}} => 可指定声音（如：Microsoft_Huihui/Microsoft_Kangkang/Microsoft_Yaoyao）
    • {{tts-voices:}} => 显示所有可用的语音列表
    • {{tts <lang>:cloze-only:<note-field>}} => 只朗读笔记字段中的 cloze 的空白部分
    • [anki:tts lang=<lang>] <raw-text> [/anki:tts] => 朗读原始字符串
  • latex => [latex] <content> [/latex] （注：需要安装 LaTeX 和 dvipng/dvisvgm）
  • {{#<note-field>}} <content> {{/<note-field>}} => 若 <note-field> 非空，那么显示 <content>
  • {{^<note-field>}} <content> {{/<note-field>}} => 若 <note-field> 为空，那么显示 <content>

标签：

  • <hr> => 分割线，如：<hr id=answer> 表示问题和答案之间的分割线
  • <br> => 换行
    注：dir=rtl 属性

实用的例子：

  • 通过 <a> 标签和卡片模板的字段替换指令，实现单词的搜索 => <a href='https://dict.youdao.com/result?word={{word}}&lang=en'>{{word}}</a> 或者 <a href='https://dict.youdao.com/result?word={{text:word}}&lang=en'>{{word}}</a>
