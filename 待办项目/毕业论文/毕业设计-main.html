内容具体要求

核心要求
内容 具体要求/解释 规模
题目 简洁明确，概括性强 不超过25个字
摘要 简明确切地描述主要内容，着重反映创新点和作者特别强调的观点，包括目的、方法、结果和结论 以150～300字为宜
关键词 反映主题概念的词或词组 一般为3～5个
正文 作者对研究工作的详细表述，包括研究对象、研究方法、结果和讨论几部分
结论 最后的总结，其内容应当完整、准确、精炼
讨论 应是由结论推广而来的，其段落通常位于结论之后
参考文献 “文后参考文献”，为撰写而引用的有关图书资料
其内容反映取材来源、材料的广博程度和材料的可靠程度，一份完整的参考文献也是向读者提供的一份有价值的信息资料
 1. 非外文专业的外文参考文献应不少于4篇，且原则上不少于总参考文献的1/5
2. 近五年文献原则上不少于总参考文献的1/4
附录（可选） 不宜放入正文但又不可缺少的组成部分，或有主要参考价值的内容
致谢 - 以简短的文字，对课题研究与毕业论文（设计）撰写过程中曾直接给予帮助的人员(如指导教师、答疑教师等)表示谢意或说明
- 这不仅是一种礼貌，也是对他人劳动的尊重，是治学者应有的品德
- 还可包括资助毕业论文（设计）的基金项目

稿子

各位老师好，我的论文题目是《基于django及pyqt5的跨平台音乐管理系统》，

  • 首先来简单介绍一下选题的背景：音乐一直以来是一种重要的娱乐和文化形式，在这个数字化时代自然就需要以一种数字化的方式取管理我们的音乐。而我着手开发这个项目原本的初衷就是为了更好的管理这种娱乐形式，随时都能带上我们的耳机听上我们喜爱的歌曲
  • 而我通过一番探索和学习呢，以及对《网易云音乐》的观察，就发现通过以下的一些技术栈就能很好的达成我们的潜在需求：我把技术栈划分为后端、桌面端、web前端、数据库四个部分
    • 其中后端通过 django 实现的数据库接口直接与 sqlite 进行通信
    • 然后这个后端又通过 DRF 来分别与桌面端的 requests 模块，以及 web 前端的 axios 模块进行交流通信
    • 使得各个模块之间互相连通
  • 然后我们来逐一介绍各个技术的用途：
  • 接着我们来分析这个系统模块的划分：本系统将操作者划分为用户、歌手、管理员三种类型，并且每个操作者对应一个子系统；然后再单独划分出一个通用功能子系统
    • 其中 用户子系统 下的 认证权限管理模块 定义了用户 注册、登录、注销，访问前端页面等操作，以及增删改查数据库的权限
    • 。。。

web 前端、桌面端、后端 的所有代码都是由我个人独立完成
